1. 虚拟表

虚拟表是 Palantir AIP 中的数据连接功能，允许开发者将现有云数据平台中的数据资产无缝注册，无需复制数据。这意味着您可以在几分钟内开始在 AIP 中基于您的数据进行构建。

在视频中，我们演示了如何建立与 Google BigQuery 的连接、注册表并立即开始构建。

有关更多详情及虚拟表当前支持的云数据平台，请参阅《文档》。

2. 嵌入

嵌入是现代自然语言处理的关键组成部分，对于部署新一代大语言模型的工作流至关重要。嵌入利用词语、短语或文档之间隐含的词汇关系，能够驱动远超传统基于词元和关键词搜索的工作流。

在视频中，我们演示了如何使用 AIP，通过嵌入模型将故障工单系统中的自由文本转换为向量，从而生成嵌入。正如下方示例所示，数值相近的嵌入在语义上也具有相似性——例如，“面膜”的嵌入向量与“面部遮盖物”的嵌入向量距离，要比与“呼吸器”的距离更近。

我们生成的向量代表了工单问题及解决方案的语义含义，便于在整个企业范围内进行知识搜索与汇总。

3. 管道构建器

Palantir 的旗舰级数据管道应用《管道构建器》，将低代码/无代码的便利性带入了一个兼具稳健性与自动扩展后端的强大平台，该后端同时支持 Apache Spark 和 Apache Flink。

《管道构建器》允许任何技术水平的用户创建生产级、可扩展的管道——同时支持对代码和管道中的数据进行类似 Git 的分支与合并操作。

我们演示了《管道构建器》如何连接来自 BigQuery 的虚拟表，利用 AI 驱动的自然语言提示构建管道清洗步骤（含文档生成），并随后通过嵌入构建器调用 OpenAI 的 ada 嵌入模型。¹

4. 基于本体的向量存储

Palantir 的《本体》位于平台核心，使用户能够为其业务和流程创建以决策为中心的数字孪生。

本体由数据、逻辑和动作构成——充当运营的实时决策图谱。作为一个语义系统，本体为大语言模型与企业安全交互提供了理想的基础。

我们演示了如何创建一个本体对象，以语义化方式表示存储在 BigQuery 中的故障工单，并展示了只需几次点击即可配置的原生向量管理功能。

5. 语义搜索

《语义搜索》允许用户突破传统的简单搜索范式，利用 AI 借助文本内在含义查找最相关的信息并驱动工作流。

在前面的步骤中，我们创建了带有语义嵌入的本体对象。这里，我们将创建一个自定义函数，作为基于大语言模型的 AI 副驾驶的工具，演示本体驱动的工具如何开启运营工作流的大门。

import { Function, Integer } from "@foundry/functions-api"
import { Objects, TitanTroubleTickets } from "@foundry/ontology-api"
import { TextEmbeddingAda_002 } from "@foundry/models-api/language-models"

export class MyFunctions {
    @Function()
    public async searchTroubleTickets(userText: string, kValue: Integer): Promise<TitanTroubleTickets[]> {

        const embedding = await TextEmbeddingAda_002.createEmbeddings({inputs: [userText]}).then((r: { embeddings: any[]; }) => r.embeddings[0]);

        let objectSet = Objects.search().titanTroubleTickets();

        return objectSet
            .nearestNeighbors(obj => obj.ticketDescriptionEmbeddings.near(embedding, {kValue: kValue}))
            .orderByRelevance()
            .take(kValue);
    }
}

6. AIP Logic

《AIP Logic》是一个无代码环境，用于构建、测试和发布由大语言模型驱动的函数。它让您能够构建利用本体的自动化业务流程，同时免去开发环境和 API 调用通常带来的复杂性。

借助 Logic 直观的界面，应用构建者可以编写提示词、进行测试、评估与监控、设置自动化等。

在这个 AIP Logic 函数（如下所示）中，我们将使用创建好的本体对象和语义搜索函数作为大语言模型的工具，以查找语义相似的问题，随后将解决方案汇总为针对当前问题的潜在步骤和解决措施。

7. AIP Assist

《Workshop》是 Palantir 的无代码应用构建框架，使团队能够快速构建复杂的交互式应用，远超传统只读商业智能的局限。所有用户操作均通过本体进行调度，旨在回写源系统并采集数据，用于包含基于人类反馈的强化学习（RLHF）的学习闭环。

在视频中，我们使用 Workshop 构建应用，利用了一个可访问由大语言模型支撑的 AIP Logic 函数的组件，该函数部署了我们创建的语义搜索功能。当用户输入关于某个问题的信息时，我们可以在本体中搜索类似的故障工单，从而基于过往的解决方案，呈现出最有可能快速解决该问题的操作建议。如此一来，我们便利用现有非结构化数据的力量来驱动实际行动——最终减少停机时间。

AIP 全程确保安全与透明，在流程的每一步都展示副驾驶的“思维链”推理过程。这使企业能够快速测试并信任各种检索增强生成（RAG）工作流。

您在本视频中看到的一切——从使用虚拟表连接和注册现有云数据资产，到创建嵌入并将其存储在本体中，再到在 AIP Logic 中创建副驾驶逻辑，最后使用 Workshop 构建应用——均在一个安全、协作的平台上完成。
