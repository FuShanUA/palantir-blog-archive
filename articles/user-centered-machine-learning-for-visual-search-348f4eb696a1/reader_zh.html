User-Centered Machine Learning for Visual Search

2024-04-09 · Original →

在之前的《博文》中，我们介绍了以用户为中心的机器学习（UCML）。Palantir 的核心 UCML 工作流允许终端用户基于自身反馈，将前沿的计算机视觉（CV）能力快速适配到特定且不断演变的任务中。它将研究级检测模型的强大能力与最后一公里定制相结合，在模型并不完美的情况下，依然能够满足用户的任务需求。

这种方法对于提升经过训练、用于检测特定类型目标的现有模型的准确度非常有效。然而，当终端用户想要识别全新的或临时的目标类型（或分析新形式的图像）时，可能并没有现成的预训练模型可用。例如，在冲突场景中，敌方力量可能会部署新的基础设施，比如供装甲车辆渡河使用的特种桥梁，或者是新型通信设备。为这些目标创建专用检测器耗时费力且需要技术门槛，因为这涉及整理标注好的训练数据集、迭代训练模型并将其部署到生产环境。在冲突期间，任务侧用户根本没有充裕的时间。我们希望他们能够独立且实时地指挥 AI，在世界各地出现新目标和模式时进行检测和跟踪，哪怕仅凭一个样本也能做到。

为了实现这一目标，我们的团队一直致力于利用视觉搜索增强 UCML，让用户能够按需发起并自助满足其定制化的检测需求。借助视觉搜索，用户可以实时检测任何图像中的任意目标，而无需针对特定目标类别构建底层目标检测模型。

通过视觉搜索，分析人员只需标出一组对应其想要检测目标的像素即可。他们还可以使用自由文本提供额外的指导和上下文。我们的 UCML 技术利用基础视觉语言模型，提取描述用户所标定像素（及文本）的独特特征，并利用这些特征在搜索集的其他位置识别相似像素。因此，用于视觉搜索的 UCML 允许分析人员实时生成针对任意图像中任何感兴趣目标的自有目标检测能力。无需数据标注，也无需模型训练。

这些视觉搜索查询可以在工作图像集上进行交互式使用，在图像档案中进行批量运行，或者部署到图像流上进行监控和预警。生成的检测结果可以与来自 Palantir 本体（Ontology）的额外数据模态和关于目标的基础元数据进行无缝融合与细化，构建出包含“是什么”、“在哪里”和“何时”的丰富视图，从而为分析和操作工作流提供支持。

下面，我们将深入解析在冲突展开、时间紧迫的情况下，用户如何在本体的支持下，利用视觉搜索生成自己的检测能力——在这个例子中是针对建筑物和车辆的检测。

用于视觉搜索的 UCML 的工作原理

下图高度概括了用于视觉搜索的 UCML 的工作原理。用户通过在图像上绘制边界框来勾勒出一组像素，指示要查找的感兴趣目标。在下方所示的示例中，用户正在寻找一架飞机，并在图像的这部分区域绘制了边界框。用户还提供了一段文本提示与标定的像素一起使用，为底层的视觉语言模型提供更多上下文——此步骤为可选，但可以为模型提供额外的特异性。

视觉搜索首先处理用户提供的边界框，将对应于感兴趣目标的像素与背景像素分离开来。在这个例子中，也就是将对应飞机的像素与机场停机坪的像素区分开。然后，视觉搜索将这些像素以及可选的文本提示输入到基础视觉语言模型中，提取出一组用于描述该目标的特征。这些视觉语言模型在训练期间已经学习了数十亿像素，能够生成高质量、详细的特征描述符。

与此同时，视觉搜索使用相同的视觉语言模型，提取用户想要搜索相似目标的每张输入图像的特征图，并利用相关性或相似度指标，识别图像每个部分与用户提供的感兴趣像素之间的相似程度。该过程的原始输出是覆盖在每张图像上的热力图，指示每个像素与用户标定的感兴趣目标的相似度。随后，利用该热力图在识别出的感兴趣目标周围生成一组边界框。

整个过程仅依赖于用户标定的区域，不需要任何独立的数据整理、标注或训练。 因此，用于视觉搜索的 UCML 不会——也不需要——对用户可能感兴趣的目标类型做出任何假设。用户在定义自己的自定义目标检测模型时拥有完全的灵活性。

虽然此示例仅涉及单个感兴趣目标，但在实际操作中，用户可以标定多个此类感兴趣目标，视觉搜索会自动进行处理。这些用户提供的感兴趣目标可以随时间存储，并根据各种元数据（如它们指示的目标类型、对应的图像类型以及地理位置）进行分组。最终形成一个共享的目标“模板”库，多个用户（或群组）可以参与贡献并不断完善定义。

用于视觉搜索的 UCML 实战应用

下面的图 2 和图 3 展示了用户如何直观地在卫星光电图像上实时生成自定义目标检测器。该示例展示了上述假设用户如何创建建筑物（图 2）和车辆（图 3）检测器。

起初，卫星图像并未显示对建筑物或车辆的检测，因为不存在用于检测这两个感兴趣目标的预训练模型。用户放大图像，只需分别在图像中围绕典型的建筑物和车辆绘制单个边界框，即可标定他们想要检测的目标（建筑物和车辆）。用于视觉搜索的 UCML 处理用户提供的单个边界框，并实时在图像上输出建筑物（图 2）和车辆（图 3）的热力图。在这两种情况下，使用的是相同的底层基础视觉语言模型。该模型也不受用户输入目标类型的限制。它只需处理用户标定的像素，就能输出准确的热力图。

结论

UCML 服务通过让分析人员在特定任务和他们正在处理的图像上下文中掌控模型，为他们带来了全新的体验。UCML 最初是为了允许用户对不完美的目标检测模型提供反馈，使其能够实时改进而开发的。借助用于视觉搜索的 UCML，我们取消了底层目标检测模型的要求；用户可以实时生成针对任意图像中任何感兴趣目标的自有目标检测能力。它不需要任何专门的数据标注或模型训练。

我们相信，这将是在普及计算机视觉能力方面迈出的一大步，使其在快速演变（且高风险）的局势中更具适应性。我们迫不及待地想看到用户将用它创造出怎样的价值。
