## 使用 Palantir HyperAuto 加速 SAP 数据集成、释放数据洞察并构建业务工作流的*分步*指南 企业资源计划（ERP）软件是大型企业的大脑。无论是营收与支出、供应商与库存、物料与制造，还是项目管理与薪酬发放，ERP 都是整个组织背后的统筹者。 任何数据团队自然都希望集成这些数据。无论目标端是数据湖、数据仓库、数据网格还是其他架构，集成 ERP 数据对于释放企业数据平台的价值都至关重要。 然而，说起来容易做起来难。 常见 ERP 背后的数据结构专为 ERP 自身的性能而设计。无论是通过 HTTP、ODBC 还是 JDBC 直连，映入眼帘的都是一片密集的命名空间、数据表和关联关系交织而成的“丛林”。 在 Palantir，我们经常看到数据团队仅凭一本数据字典和数据库命令行工具，就在这片“丛林”中艰难跋涉。工程师们往往要在周末发布代码，祈祷在测试环境中编写的查询语句在生产环境运行时，不会导致系统锁死进而瘫痪整个公司。 退一步讲，就算数据完成了集成，还需要持续保持数据更新且不出错。构建一条具备错误检测能力的高性能增量管道，绝非易事。 这也难怪企业常常寻求引入外部专家的支持。 另一个选择是 Palantir HyperAuto。借助这款智能软件应对 ERP 系统的复杂性，数据团队就能腾出手来专注核心业务。在接下来的教程中，我们将展示如何将耗时数月的数据集成工作，压缩到一小时内完成自动化。 ## 那么，HyperAuto 究竟是什么？ HyperAuto 能够连接并集成来自 ERP 和 CRM 系统的数据。它会分析 ERP 的元数据，从而生成健壮且可扩展的数据管道。这些管道通过自动创建的数据转换流，对业务对象进行反范式化重构，这意味着你无需再精通专有的 ERP 表结构。 随后，HyperAuto 可以将集成后的数据推送至多种云平台，例如 Google BigQuery、Snowflake 平台、AWS RedShift 等等。或者，只需点击两下，就能将数据推送到 Foundry 本体。本体会将数据组织为易于理解的语义对象，组织内的任何人都可以自助使用并做出业务决策。 ## 数据团队选择 HyperAuto 的五大理由 **1. 安全且大规模地集成数据** HyperAuto 能够大规模摄取数据，并自适应其独特的数据形态。它支持从本地和云端来源进行批量及增量摄取。借助完善的访问控制，你可以确保合适的人员在合理的用例下访问合适的数据。内置的数据治理工具让访问审计变得轻而易举，并且能够模拟权限变更。 **2. 针对最常见企业系统内置连接器** 减少配置时间，将精力投入到执行上。HyperAuto 内置了对 SAP、Salesforce、NetSuite 和 HubSpot 等行业标准 ERP/CRM 系统的支持。 **3. 针对其他数据源的灵活工具链** 数据集成往往是一团乱麻：并非所有源系统都能完美套用千篇一律的解决方案。HyperAuto 具备扩展性。只要你的系统存在实体关系，它就能将其反范式化为可直接用于分析的格式。基于 Palantir 的数据连接技术，HyperAuto 支持云文件系统/对象存储、流数据源、JDBC 数据库、REST API、NoSQL 存储等。你可以在《Palantir 文档：源类型概览》中探索完整的连接器列表。 **4. 快速上手且不牺牲稳定性** 几分钟内，HyperAuto 的软件定义数据集成（SDDI）套件就能生成 Spark 代码，用于清洗、转换和标准化你的数据。最终生成的是人类可读的代码，并配有一个集成开发环境，可在同一处进行审查、修改、测试和版本控制。所有代码都会检入 git 版本控制系统，因此可以像管理人工编写的代码一样对其进行管理。 **5. 集中管理数据并推送至你选定的目标系统** 一切数据都处于随时可分析的架构下，你可以将其加载到任何所需之处。使用 HyperAuto 的数据导出工具将其发送至数据仓库，或利用 Foundry 的本体从零开始快速挖掘洞察。 ## 教程：使用 HyperAuto 集成 SAP 数据 我们通过一个例子来说明 HyperAuto 的独特之处。在此场景中，假设你是一家制造企业的数据工程师，希望在不同源系统间构建供应链的完整视图。作为起步，我们将使用 HyperAuto 从 SAP 摄取供应链数据，进行清洗，然后将其导出至 S3 存储桶。需要说明的是，下文涉及的数据均为虚构数据。 虽然此处我们专门针对 SAP 集成进行讲解，但你可以在《Palantir HyperAuto 官方介绍》中阅读关于 HyperAuto 其他应用的更多信息。 **#0：设置 SAP 连接** 首要之事：前往文档中心，通过官方 SAP 认证的连接器将你的 SAP 数据连接上线。有关入门的更多信息，请点击《Palantir 文档：SAP 源类型》。 **#1：访问你的 SAP 实例** 现在，我们将通过“数据连接”应用访问已连接的 SAP 实例。你可能有好几个集成，因此请务必寻找类型为“magritte-sap-source”的源，并定位到对应正确 SAP 实例的源。有关“数据连接”布局和命名的更多信息，请点击《Palantir 文档：数据连接概览》。 **#2：探索你的数据** 点击“探索并创建同步”以打开 HyperAuto 的探索界面。HyperAuto 会按需直接从你的 SAP 实例读取数据，因此它将包含 SAP 的标准表以及你创建的任何自定义表。在这里，我们可以通过类似可视化购物车式的流程，挑选并确定要导入的数据表。 更重要的是，由于你能够准确查看并选择所需的数据，从而避免了不必要的昂贵查询开销，最大程度减少对 ERP 实例的潜在干扰或停机风险。 **#3：使用预置工作流开始导入数据** 在此界面你有几个选项，比如导入单个 SAP 业务对象，但我们将先从“工作流”选项卡中导入一个 HyperAuto 预置的工作流。我们将点击“供应链中断”工作流旁边的“+”按钮将其导入。这会自动将该工作流涉及的表包含到你的探索区域中。按下“布局”按钮可获得更清晰的图形化展示。该可视化图表能让你非常直观地看清所有表之间的关联关系。 HyperAuto 配备了预定义的行业用例或工作流，例如供应链中断和库存管理。当使用特定工作流时，HyperAuto 会自动提取该工作流所需的业务对象和表，并在摄取数据时应用必要的逻辑，使数据直接满足你的业务场景需求。 **#4：通过额外的表自定义导入内容** 除了这些工作流之外，你还可以使用所需的任何表进一步自定义导入内容。例如，假设你被要求包含度量单位的元数据。导航到“模块”选项卡，搜索物料的度量单位表（MARM），然后点击“+”按钮将其包含到你的探索区域中。 **#5：探索表间关系以找到所需数据** 提前预知所有需要的表并非易事，因此 HyperAuto 可帮助你探索 SAP 实例。即使事先对 SAP 的表结构一无所知，你也能轻松在相互关联的表之间导航，发现更多可用数据。在后台，HyperAuto 会将所有内部表名和列名翻译为人类可读的形式。你可以将其视作自动化的数据字典。 在探索这些表时，我们看到“物料描述”引用了一个客户表，因此我们也一并导入该数据。点击“物料描述”上的链接图标以显示其相关表，然后右键点击客户表并选择“将 1 个表添加到同步”将其纳入导入范围。 **#6：配置同步** 点击“配置同步”以审查拟定的数据摄取方案。在此，你可以微调设置并预览各个表的数据。这赋予了你调整摄取策略的灵活性；例如，你可以选择快照或追加模式。这让你可以选择是完整摄取数据还是仅拉取增量记录。你可以在《Palantir 文档：数据集成与事务类型》中查看更多信息。 在本教程中，我们将保持默认设置，依次点击“摄取并集成数据”按钮和“创建同步”按钮予以确认。 **#7：生成管道代码** HyperAuto 现在将配置与源数据库的所有摄取同步。它还会编写管道所需的所有 PySpark 代码。魔法吗？不尽然：通过解析系统内的实体关系，我们能够重建业务对象，随后将“供应链中断工作流”中的逻辑叠加其上。 PySpark 是一个具备极高扩展性和强大功能的数据转换框架，其内存计算机制提供了卓越的性能。由于所有代码都通过 git 进行版本控制，你可以像处理人工编写的代码一样对其进行操作。 随后你会看到一个弹窗，提示已为这些更改提交了拉取请求。由于所有更改都存在于分支上，你可以在合并至主分支之前执行组织要求的任何检查和审查。现在，我们将让该拉取请求在后台自动合并。点击“下一步”继续，然后关闭此窗口。点击“导航至 SDDI”进入 SDDI 界面。 **#8：运行生成的原始数据摄取任务** 逻辑准备就绪后，接下来就是引入数据。你可以按照 SDDI 流程的引导完成数据集成。作为第一步，我们需要从 SAP 摄取原始数据。点击“后续操作步骤”以运行我们刚才设置的同步，以及后续所需的元数据同步。然后，HyperAuto 将在后台开始拉取所有数据。 **#9：打开管道可视化界面** 完成后，点击“生成管道”进入下一步。此页面提供了有关管道所有待处理更改的全局视图。如果有多人协作并决定在步骤 7 中增加额外的检查环节，我们可以在此处浏览这些内容。由于我们的拉取请求已经合并，请点击“打开数据血缘”以在数据血缘中打开该管道。 **#10：构建清洗与集成管道** 数据血缘展示了原始表和转换后的表之间的相互关联。图上的每个矩形节点代表一个数据集及其生成代码。连接线代表数据帧的输入和输出，这让深入探究转换之间的依赖和关联变得十分简单。此页面集成了丰富的洞察信息，你可以在同一处快速预览每个表的数据和代码、追溯用户权限以及审查构建性能历史。现在，我们将利用它快速构建新的管道。 如需继续，请通过“选择 → 选择所有”选中所有表，然后点击构建图标并按照步骤操作。 **#11：审查最终数据** 此操作将运行 HyperAuto 生成的清洗和集成步骤。这些构建完成后，你可以预览数据并查看最终数据集背后的代码。 **#12：创建 S3 同步源** SAP 数据现已完成清洗和集成，下一步是将其推送至你的业务操作层。在本例中，我们将连接并推送数据至 S3 存储桶。要设置此连接，请返回“数据连接”应用，选择“新建源”，然后选择“S3”。在此处，你需要通过选择运行此导出任务的代理并填写 S3 存储桶的详细信息来配置连接。你必须将新的数据连接源保存到某个项目文件夹中，以便对其进行正确的权限管理。 **#13：创建数据导出任务** 现在，我们将在此 S3 源上配置一个任务，以导出我们处理好的数据。导航至你保存源的文件夹，右键点击它并选择“创建数据连接任务”。在此窗口中，我们将配置任务的输入、输出和设置。 在配置窗口中，我们将任务类型设置为“export-s3-task”，并声明 S3 存储桶内的输出目录路径。虽然还有存储桶策略和重写策略等其他选项可以修改，但目前我们先保持默认设置。 接下来，我们将配置要导出的表。在“输入”下点击“添加条目”，导航至最终确定的数据表并添加你希望导出的内容。最后，通过在“输出”下点击“添加条目”并选择一个输出文件夹，创建一个用于捕获导出结果的输出。 **#14：运行导出任务** 要运行导出，请点击输出数据集，然后点击“构建”。在后台，我们配置的表将被导出至 S3 存储桶。未来，该构建可以配置为在底层数据更新时自动运行，或按照设定的时间表执行。 完成这些步骤后，你已经导入了多个 SAP 对象，对其进行了清洗，将其反范式化为可用于分析的数据集，并将其导出到了数据仓库。接下来，你可以使用这些数据集来支撑任何可视化或分析工具，例如 Tableau 或 Power BI。或者，你也可以将数据推送到 Foundry 自带的分析和工作流工具中。选择权在于你。 现在你拥有了一项可扩展的数据资产。使用 HyperAuto，你可以将来自多个 SAP/ERP 系统的对象进行组合，并利用来自其他源（如时间序列或地理空间数据）的数据对其进行丰富，从而为你的业务构建更全面的视图。 ## 结语 在这篇文章中，我们探讨了如何使用 HyperAuto 轻松、快速地将数据从 ERP 系统中提取出来，并输送到你选定的目标端。过去耗时数月甚至数年才能完成的复杂数据集成项目，现在只需几小时即可搞定，这让工程师能够专注于真正重要的事情：以人们易于使用的格式，将数据交付给需要的人。 想了解这一方案在实际落地中的成效吗？不妨了解 Fujitsu 如何在几天内集成各种 ERP
