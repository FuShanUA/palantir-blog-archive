Building with Palantir AIP: The Ontology Software Development Kit

作者：Chad Wahlquist，Palantir 前线部署架构师

欢迎来到《使用 Palantir AIP 进行构建》系列的又一集，Palantir 的工程师和架构师将带您了解如何使用我们的《人工智能平台 (AIP)》构建端到端工作流。在这里，我们将深入探讨本体软件开发工具包 (OSDK)。

引言

我们将探讨 OSDK 如何让您能够为企业构建自定义 SDK，直接从开发环境中充分利用《Palantir 本体》的全部功能。借助 OSDK，您可以快速无缝地将定义企业的数据、逻辑、动作以及大语言模型 (LLM) 集成到现有应用程序中，创建全新应用，与后台系统对接等等。

我们将首先定义 OSDK 以及它如何融入更广泛的平台生态系统。然后，我们将在虚构的 Titan Industries 本体上生成自己的 SDK（探讨安全、平台内 Web 托管和自动文档生成）。最后，我们将把 SDK 集成到一个用于卫星任务调度的 React 应用中。

开始构建吧！

用例

对于看过我《上一个视频》的人来说，我们的用例并不陌生——使用 AIP，我们为虚构的医疗用品公司 Titan Industries 构建了一个应用，帮助 Titan 在其某个配送中心发生火灾后应对供应链紧急情况。该应用使我们能够呈现受火灾影响的配送中心的客户订单，识别拥有足够库存来填补订单的备用配送中心，并相应地更新这些订单。

现在，我们想使用 AIP 来帮助我们评估该配送中心的损坏情况。为此，我们将扩展已经构建好的 Titan 本体，使其包含对不同公共卫星的任务调度，并使用大语言模型来寻找合适的卫星以捕获配送中心的图像。

然后，我们将为 Titan 本体创建 SDK，并将其集成到一个 React 应用中，该应用将帮助我们捕获所需的卫星图像。

什么是 OSDK？

Palantir 本体将企业的数据、逻辑和动作整合到以决策为中心的企业模型中。本体 SDK 就是企业的 SDK，使用的是企业的语言。

正如本体系统产品负责人 Peter Wilczynski 在其《面向本体的软件开发》博文中所述：

“本体集中了这些[业务]知识，并将其封装在一个共享系统中。该系统与 OSDK 一起，构成了编写业务逻辑的更高级抽象，运行在由关键业务概念、运营流程和真实世界任务构成的统一层上。这样，从以组件为中心的表示法到共享概念模型的转换只需进行一次，而不是每次构建新应用时都要重新转换。

而且这种知识会迅速累加；通过依赖以决策为中心的共享本体，新应用可以利用现有的系统集成工作，通过将新组件的数据、动作和逻辑元素与现有表示相统一，可以快速接入新组件。”

但在实践中这意味着什么？为了说明这一点，我们将演示在开发者控制台 (Developer Console) 中创建一个演示应用，这是一个帮助我们在已构建的本体 SDK 基础上开发应用的门户。我们能够在本体之上创建多个 SDK（例如，用于自定义集成、移动应用等），并且在整个过程中严格遵守本体中设定的安全和权限。

应用类型

开发者控制台支持两种类型的应用：

  1. 面向客户端：前端服务，包括 Web、桌面或移动应用。出于安全原因，面向客户端的应用设计为不具备存储客户端凭证的能力。
  2. 后端服务：用于支持后端工作流的应用，例如应用服务器、守护进程或集成。这些应用可用于安全地存储客户端凭证。

我们可以根据应用的架构方式选择其中一个或两个选项：通常，如果我们要为已有登录账号的用户构建应用，可以使用面向客户端的应用类型。在选择不同选项时，开发者控制台会通过真实案例向我们展示这些选择对整体安全态势的影响。

权限

创建应用时，必须在两种用于访问应用数据的权限类型之间进行选择：

  1. 用户权限：用户的权限将决定其可对本体执行哪些读写操作。这意味着，我们从安全和权限角度构建的任何内容都会得到严格遵守——在构建自定义应用时，无需重建安全和权限方案。
  2. 应用权限：应用将使用生成的服务用户进行权限控制，而不是应用用户权限。

同样，在选择不同选项时，开发者控制台会通过真实案例清楚地说明整体的安全影响。

本体与资源作用域

接下来，我们选择相关的本体和组织，然后是关联的对象、动作和函数。

我们可以逐个选择对象，仅提供应用所需的访问权限——避免意外暴露链接、动作和函数。

随后，我们可以添加不同的动作（在此例中为创建工厂事件）。关键在于，这些动作使用的是业务语言；这使我们能够通过回写源系统并在企业内触发不同动作，有效地实现“闭环”。

暴露函数（包括由大语言模型支持的函数）非常简单；在此例中，我们引入了在《AIP Logic》中构建的卫星任务调度函数，以帮助我们在应用中找到正确的卫星。

得益于 OSDK，将回写、大语言模型和其他形式的生成式 AI 集成到新应用和现有应用中，并不意味着要从头开始构建；我们不仅能轻松引入对象，还能轻松引入动作和 AI 支持的函数。

审查

在审查时，开发者控制台清楚地指出了我们后续能够和不能够更改的内容。例如，组织和本体后续无法更改，但我们可以在事后添加更多对象、链接、动作和函数。

生成您自己的 SDK：Titan 卫星任务调度
回到我们的卫星任务调度：为了实现使用卫星图像评估配送中心损坏情况的目标，我们将构建一个利用 OSDK 的 React 应用，该应用将访问 Titan 本体和由大语言模型支持的函数，帮助我们找到合适的卫星。

让我们来看看将要构建 SDK 的本体（下图）。蓝色代表我们用于供应链管理应用的现有 Titan 本体。绿色表示我们为 SDK 添加的新对象和动作类型：工厂事件（例如最近的火灾）；传感器任务（在特定位置提供我们所需的图像类型）；以及卫星（能够捕获这些图像）。

逐步增加本体内容简单、快速，并为整个企业带来累加效益。在单一平台上构建和集中管理本体的能力意味着，我们可以专注于应用构建，并减少构建和维护数据基座通常所需的负担。

现在回到我们的 SDK。我们已经有了 Titan 卫星任务调度应用 SDK，我们已根据需要为其提供了对本体中不同对象、链接、动作和函数的访问权限——而且我们可以轻松添加更多。SDK 不是静态的；它们可以根据用例的需要进行演进。

该 SDK 可以创建为用于 TypeScript 的 NPM (Node Package Manager) 包，或者用于 Python 的 Pip 或 Conda 包，甚至是 cURL 调用。我们可以选择要使用的包生成器的特定版本。

开发者控制台还提供平台内全面管理的 Web 应用托管功能。这意味着希望快速构建基于本体的应用（并具有强类型安全性和安全性）的开发者，可以在平台中构建、托管它们，并通过移动端、内联网或 Web 提供访问。

开发者控制台会根据我们选择的本体实体自动生成文档。该文档支持 TypeScript、Python 和 cURL（我们可以使用下拉菜单轻松在不同语言间切换）。当我们更新本体时，此文档会自动更新：我们拥有一个能自动记录我们的本体及与之交互的各种方式的 SDK。

使用 OSDK 构建 React 应用

现在我们准备好构建并集成我们的应用了。在 VS Code 中，我们安装了 SDK，并获得了一个 README 文件，其中包含了运行它所需的步骤。

在编写代码时，我们可以轻松利用通过 SDK 暴露的任何内容。我们可以从文档中复制并粘贴代码，按需引入——例如，加载配送中心或调用卫星任务调度大语言模型函数。

有了这个，我们能够快速创建一个 React 应用，在地图上呈现配送中心列表，以及可用卫星和相关位置。我们可以选择一个配送中心——在此例中为最近受火灾影响的 Haynes 配送中心——并调用我们由大语言模型支持的函数，以寻找一颗将经过该配送中心并能够捕获我们评估损坏所需图像的卫星。

当函数选定卫星并按下“保存”时，它会回写到 Titan 本体并创建一个任务对象——因为任务是我们 SDK 的一部分。我们可以立即在本体中看到刚刚在应用中创建的任务对象，包含卫星、链接、适当的权限等完整信息。
