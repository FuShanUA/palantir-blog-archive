Implementing MOSA with Software Defined Systems Integration

2024-04-05 · Original →

TITAN 介绍

为应对未来快速演变的威胁与复杂对手，保持竞争优势，国防部正积极拥抱并大力推行软件主导的硬件，这在陆军最新的现代化项目中可见一斑。Palantir 很荣幸能参与并引领这一变革浪潮，这从近期的 TITAN 项目采购以及“机器人战车”（RCV）降阶选型中便可见一斑。

新兴技术驱动的国防部新系统所需的关键差异化优势，在于增强的模块化、开放性和灵活性。随着软件在获取非对称优势中的核心地位日益凸显，我们 Palantir 坚信，这种模块化、开放式的软件将成为一项关键优势，使未来的地面、空中、海上和太空能力得以实现并保持领先地位。

然而，如何观察或衡量一个系统的模块化和开放性，似乎既是技术挑战，又带主观色彩。本文将探讨一种重要方法——MOSA，并举例说明 Palantir 如何基于行业最佳实践和数十年的实战经验来设计解决方案。

为何 MOSA 至关重要

在深入解析上述两个项目之前，有必要先了解国防部的《模块化开放系统方法》（MOSA）对未来项目何等重要。在快速演变的威胁环境中，MOSA 允许复杂系统的各组件单独竞标，从而采购到经济可承受、适应性强且能力出色的系统；同时，它还能让这些系统持续汲取技术创新，在与同量级对手的较量中保持竞争力。

在 Palantir，我们从底层架构起就将 MOSA 原则融入到面向政府的解决方案中。我们的软件产品（包括部署在 TITAN 和 RCV 中的产品）由多个功能模块组成，这些模块具备高内聚、低耦合、可扩展、可互操作且完全可拆分的特性。这确保我们既能增强现有系统并在当下端到端解决方案中交付成果，也能实现高度特定的功能，或是兼顾两者间的任何需求。

Palantir 的 MOSA 实践

Palantir 的解决方案构建于 MOSA 框架之上，提供的功能使得在复杂的陆军解决方案生态系统中，部署创新技术变得更容易、更具成本效益且更迅速。

Palantir 通过我们研发的“软件定义系统集成”（SDSI）方法来管理设计、管理与升级的复杂性，这正是 MOSA 原则的一种落地实现。SDSI 方法有助于识别关键系统架构组件、开发客户自有的关键系统接口与数据规范，并以代码方式管理“系统的系统”的复杂性。

借助 SDSI，用户可通过以下四种方式调整和增强模块化系统：

1. 替换：用新组件替换原组件的功能

系统可动态替换组件，包括硬件组件、软件包和算法。在 TITAN 项目中，我们大量进行各种 AI 模型的轮换测试，以确保在任何运行条件下都能达到最优性能。传统系统只能在设计阶段更新，而 SDSI 则支持在系统上持续执行替换操作，甚至在系统运行期间亦可进行。

2. 扩展：通过修改现有组件或引入新组件来增加功能

现代硬件由数十家供应商的解决方案构成，其中许多供应商通过 API 和系统接口为系统添加新功能（例如新算法或特定领域的分析工具）。

SDSI 使得系统能力得以成熟并拓宽应用场景。最终，这让用户能够在功能演进时，以低成本、高速度将同类最佳的能力添加到解决方案中，同时不影响现有功能。

3. 导入：整合来自其他位置或系统的数据

利用符合政府和行业数据规范的灵活数据集成工具，系统几乎可以摄取并处理任何格式的数据。SDSI 坚持系统中可处理数据的数量与多样性，最终为战场构建强大的解决方案。

4. 导出：以正确的消息格式将数据从系统发送至另一位置

SDSI 允许对系统进行快速配置，以准确、快速、高效地将数据和信息发送至执行器。为达成目标，用户需获取洞察并得出结论，同时将信息发送至其他系统，以实现完整的情报与瞄准全生命周期。

这些理念基于 Palantir 为客户设计开发模块化开放系统的经验，并与政府及企业的最佳实践和标准保持一致。我们已实施并运用了政府、开源和商业标准与接口，并为政府开发了特定项目的 API 和数据模型。因此，所有关键组件和接口都允许用户在项目的整个生命周期内利用并扩展 Palantir 的软件。

TITAN 介绍

TITAN 是陆军的新一代深度感知情报地面站，由人工智能和机器学习赋能，代表着首创性的现代化项目。TITAN 的首要目标是提升陆军的战场远距离感知与快速决策能力，旨在替代多种传统工具和系统，在提升能力的同时，降低操作员的操作复杂度与时间成本。

TITAN 项目展示了 Palantir 如何为陆军情报界设计并交付一套软硬件集成系统。该系统旨在尽快列装，同时保持适应能力，以确保在未来数年内持续发挥效用。在当前形态下，TITAN 连接了涵盖太空、空中、陆地和海洋领域的数十个数据源与系统。它贯穿陆军的多个层级，并跨越绝密、机密、联军和涉密网络环境运行。

Palantir 的 TITAN 系统整合了来自数十家公司和政府机构的组件与解决方案，涵盖人工智能、数据管理、数据关联、情报分析、地理空间分析、通信、电源管理等诸多领域。

TITAN 演示的关键亮点包括：

  • 展示了关键组件的模块化，如 AI 探测模型、传感器融合算法、地图组件、特定领域的 GOTS 和 COTS 工具，以及数据集成逻辑。
  • 开展并记录了数十个软硬件子组件在相对性能与成本方面的权衡研究。
  • 基于我们的开放式设计开发了实施原型，缩短了配置、设置、运行、管理和撤收的时间周期。
  • 在降低终端用户复杂度和认知负担的同时，提升了相较传统系统的探测与识别范围。
  • 与陆军及合同合作伙伴协同开展了快速迭代的“测试-修复-测试”流程。

Palantir 在 TITAN 项目上的工作，是项目采购与执行“软件路径”的成功范例。作为软件主承包商，我们期待继续努力，为全球陆军用户提供支持。

下一前沿：RCV

经过竞争性的“商业解决方案开启”（CSO）降阶选型流程，Palantir 被选为软件系统集成商，以支持“地面车辆自主路径”（GVAP）项目，为“机器人战车”（RCV）项目原型集成包括安全关键软件在内的各种异构软硬件技术。

通过美国陆军与国防创新部（DIU）的合作，Palantir 及其他公司将开发自主软件与流程原型，将无人车辆技术适配到多种极具挑战性的军事环境中。该项目将建立一种能力，实现将自主功能与载荷快速集成到地面战车中。值得一提的是，该项目是陆军首个将硬件和软件采购路径分离的项目，也是首个采用软件集成商的项目。

Palantir 借助“软件定义系统集成”的方法，将确保在硬件和软件两方面都取得比传统方法更出色的性能与更高的可靠性。这种从互不相连、孤立部署的模式转变，将拓宽整体系统的性能边界，并在系统演进时实现快速创新。

Palantir 的解决方案弥合了第三方软硬件原型之间的鸿沟，通过模块化、可拆分的软件架构，确保同类最佳能力的集成以及 RCV 项目未来的可扩展性。依托一套 Palantir 操作系统，SDSI 方法赋予了 RCV 项目对原型开发工作的可见性，并在战场及随之而来的需求持续演变时保持灵活性。

强大的软硬件集成系统

作为首家软件主导的主承包商，Palantir 展示了如何将数十个软硬件组件——代表着来自商业界、政府和学术界的同类最佳能力——进行快速集成、测试和评估，从而缩短未来任务中从传感器到射手的工作流时间。Palantir 经过验证的 SDSI 软件解决之道实现了真正的模块化，带来了从原型设计到生产阶段无与伦比的灵活性、可扩展性和对工作的可见性。
