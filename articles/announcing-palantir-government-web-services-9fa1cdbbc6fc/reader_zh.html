Technical Annex: Announcing Palantir Government Web Services

Palantir 政府网络服务（PGWS）旨在将我们过去 20 年间为开拓和扩大政府业务而开发的基础构建模块，作为独立、单一的提供项对外发布。就像超大规模云服务商让互联网企业能够以更快的速度、更低的成本构建应用一样，PGWS 也将借助驱动 Palantir 各平台且已获认证、合规并经过验证的技术，助力政府技术创新者走得更远、跑得更快。我们希望通过利用我们为打入政府领域而构建的基础设施，使客户与合作伙伴能够将精力集中于国家所需的下一代开发与创新上。我们的前三项 PGWS 提供项是 FedStart、Apollo 和 Ontology SDK。

FedStart 与 Apollo：消除认证壁垒

DevSecOps 相较于国防部在 2003 年的状态已是巨大进步，但力度仍显不足。我们构建 Apollo 作为符合 DevSecOps 标准的平台，使我们的工程师能够跨众多异构环境与网络，持续交付并自主部署现代软件。大多数商业公司只需在云端运行少数几个大型、按地域划分的软件实例即可。而面向国防部的解决方案，则需要在众多物理隔离环境下——跨越边缘与云节点、网络及资产——持续部署数十乃至上百个微服务。

现有解决方案无法应对这一挑战。Apollo 的自主 DevSecOps 方法将服务与环境解耦，在不扩充 DevOps 规模的前提下实现实例扩展。洛克希德·马丁公司正使用 Apollo 对宙斯盾系统进行现代化改造。Anduril 正在长弓系统上评估 Apollo。思科公司正利用它来标准化跨商业与政府环境的软件交付方式。Apollo 使得以真正具备规模的 SaaS（软件即服务）形式向政府客户部署和管理软件成为可能，而无需按比例线性增加高密级环境的站点可靠性工程师或 DevOps 人员编制。

Apollo 与现有的软件开发基础设施及制品库集成，让将软件部署到 SC2S、C2S、物理隔离的边缘节点、车辆和传感器上的过程变得轻而易举。它实现全生命周期管理的自动化，包括将二进制文件迁移至网络、编排部署、回滚、安全以及漏洞管理。

除 Apollo 外，我们还提供 FedStart——一项获得认证的平台即服务。与国防部合作要求每个生态合作伙伴为其每个软件实例取得认证。说服发起方提供认证往往耗时费力，而一旦合作伙伴加入，他们还得准备平均花费 18 个月和 200 万美元来完成认证流程——这对大多数明智的 VC 支持的公司而言并不可行。借助 FedStart，将容器化产品引入我们 PaaS 的公司能迅速向客户提供 IL5 级 SaaS，并将于 2024 年 1 月起提供 IL6 级 SaaS。FedStart 将原本长达 18 个月、耗资约 200 万美元的认证流程，缩短至最快 6 周内完成，且成本仅需一小部分。

Ontology 软件开发包（SDK）：消除数据访问壁垒

我们能为国防技术生态赋予“非对称优势”的第二个切入点，是通过 Ontology 软件开发包（OSDK），让其获得已获授权的政府数据访问能力。过去 20 年中，我们面临的最棘手障碍之一，就是需要费尽周折去对接国防部各源系统。缺乏统一流程，使得用统一程序解决此挑战的愿景始终看似触手可及，却从未真正落地。结果就是我们只能自己动手，一点一滴去实现。

数据连接：总计，我们已集成 1,150 个数据连接源，以及与作战司令部及各军种相关的近 24,000 个同步任务，涵盖作战、人事、后勤、运营、情报、编制、训练及装备系统。我们的软件提供的不仅是行列数据的简单倾倒，而是一种面向对象的世界观——即本体论——反映出哪些数据具有价值及其相互关系。通过高度符合人体工程学的 Ontology SDK 和 API 即可获取此类洞察。

OSDK 面向谁？拥有对相关项目已获授权访问权限的政府项目、国防公司及个人开发者，能够使用 OSDK 让其应用安全地检索、获取并丰富关联数据。这使客户免受与构成任何单个项目（如陆军 Vantage 系统）的数十乃至上百个源系统对接的折磨。

仅需四行代码，国防技术生态即可充分利用我们 20 年数据集成所沉淀的全部数据。

超越数据连接：OSDK 不仅简化了数据连接，还通过以最少代码访问本体，实现应用快速开发。更优的开发者体验意味着更好的应用以及与一线用户更快的反馈闭环。“数字化转型”正是由解决具体问题带来的一个个成功故事和成果汇聚而成，无论是利用生活模式算法进行船舶追踪，还是利用预测性分析研判乌克兰战场弹药消耗速率。OSDK 赋予开发者和操作人员解决此类具体问题的能力，且无需端到端使用 Foundry。我们已将 Foundry 本体剥离出来，并支持在 React 应用中轻松使用 OSDK，从而对前端和后端实现完全掌控。

规避厂商锁定：OSDK 在顶层（应用层）和底层（数据层）均消除了厂商锁定。在顶层，可以开发并部署通过 Ontology API 实现互操作的新应用。这与使所有 Palantir 产品实现互操作的机制相同，且我们已将该能力扩展至任何商业及政府构建的应用。在数据层，脱离 Palantir 的迁移成本仅限于让其他供应商或政府项目使用 API 将数据拉取至其平台。

即将推出

用于操作复杂软件的 Witchcraft：我们下一个开源项目是 Witchcraft，这是我们自 2016 年起在内部使用的工具，用于提供零配置应用服务器。它在日志、SSL、调用追踪、HTTP/RPC 语义、应用配置及身份认证方面默认采用最佳实践，从而简化了运维挑战。我们将提供简化合规性并降低门槛的审计日志库，以及一套在线数据库升级框架，以减轻开发者的繁琐劳动，并使在更复杂的环境中操作复杂软件变得切实可行。作为额外福利，Witchcraft 服务可自动与 Apollo 和 FedStart 协同工作。

访问控制与数据共享：Palantir 在 9/11 事件后不久成立，旨在为情报界提供产品。从第一天起，我们核心产品的一部分就是安全共享数据的能力。这意味着我们所有的产品在设计之初就将安全和数据保护作为基础原则，而非事后补充。作为路线图的一部分，我们希望将安全能力作为独立的授权服务提供给生态系统，包括：支持基于密级的访问控制、基于角色的访问控制和细粒度访问策略（其中策略针对数据内容进行唯一性评估）的认证与授权技术；支持促进跨合作伙伴的安全共享，这对于与盟友的有效协作至关重要；以及符合国防标准的用户和组管理，如 CAC/PIV 认证、权限验证与审批等。

未来

上述能力仅是我们计划向国防技术生态提供内容的开端。我们致力于让客户和合作伙伴在需要时，能够利用我们在为政府交付过程中积累的来之不易的经验，从而将精力集中于下一代的开发上。

提供正式工具以赋能生态，是我们长期秉持观点的加速与延伸，并建立在我们的开源贡献与项目，以及我们对无人机公司推广开放标准、提供指导和支持的基础之上。这一理念塑造了我们提交的每一行代码：从我们即插即用的零信任 DDIL 应用解决方案的未来，到我们在混合现实领域的投资。在 Palantir，每一项投资和路线图不仅通过开放的视角被重新审视，更以加速整个生态为目标，进而为“第一顿早餐”的餐桌前创造更多席位。
