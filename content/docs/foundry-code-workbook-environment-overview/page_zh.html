环境

每个代码工作簿都关联着一个环境。环境包含一组 Conda 包和 Spark 设置，它们安装在为工作簿提供计算支持的 Spark 模块上。打开工作簿时，在工作簿获取 Spark 模块的过程中，您可能会看到等待资源或正在初始化环境的提示。在获取到 Spark 模块之前，您无法使用控制台或运行转换操作。

每位用户都会被分配一个 Spark 模块，该模块可供同一项目中且使用相同环境的各个工作簿共用。Spark 模块绝对不会在不同用户之间共享。

选择配置文件

您可以点击环境，然后点击自定义 Spark 环境来配置您的环境。此时您会看到一系列《预定义配置文件》。管理员已将这些配置文件配置为适用于特定工作流或用户组的默认设置集合。

若要使用预定义配置文件，请在左侧面板中选中它，然后点击更新 Spark 环境。

修改配置文件

如果预定义配置文件都不符合您的使用场景，您可以通过在控制面板中编辑预定义配置文件，或者在工作簿中自定义预定义配置文件的方式，添加或移除包来修改配置文件。这两种方式的主要区别在于修改的生效范围。若在控制面板中编辑预定义配置文件，所有使用该配置文件的工作簿中安装的包都会相应发生改变。相反，若在工作簿中自定义预定义配置文件，仅该特定工作簿中安装的包会发生改变。

要在控制面板中编辑预定义配置文件，您可以《为代码工作簿配置文件配置包》。

要在工作簿中自定义预定义配置文件，请打开目标工作簿，点击环境，选择配置环境，然后点击自定义配置文件按钮。在自定义视图中，您可以点击减号来移除现有包，或更改其请求的版本。要添加新包，请在包侧边栏中进行搜索，然后点击加号。将其添加到配置文件后，您可以选择特定版本、选择AUTOMATIC，或指定自定义 Conda 版本。

自定义环境的初始化速度会比预定义配置文件慢。

添加配置文件

若要创建供各工作簿使用的新预定义配置文件，您可以在控制面板中《配置代码工作簿配置文件》。

自定义 Conda 版本

自定义 Conda 版本可以是：

  • 一个版本号（例如 3.6）
  • 一个比较运算符加上一个版本号（例如 >=3.6）（可接受的比较运算符包括 =、==、>=、>、<= 和 <）。有关更多详细信息，请参阅《Conda 包规范 ↗》。
