共享 Python 库

跨多个 Python transforms 仓库共享代码的推荐工作流是发布一个 Python 库包，具体来说就是 Conda ↗ 库。1.23.1 及更高版本的 Python transforms 仓库支持发布 Python 库。

发布 Python 库

发布 Python 库需要以下步骤：

  1. 创建一个新仓库，用于存放共享库的 Python 代码。

  2. 为仓库命名。初始化时，库的名称将与仓库名称保持一致。其他代码仓库将通过该名称来发现并使用您的库。后续如需重命名，可以访问 gradle.properties 文件并编辑 condaPackageName 参数（该文件为隐藏文件，您可能需要先在文件编辑器中选择“显示隐藏文件”）。

库命名

请注意，condaPackageName 只能包含 ASCII 小写字母、数字或连字符。任何非字母数字或非连字符的字符序列都将被替换为单个连字符（例如，my_library repo 将发布为 my-library-repo，而 Foobar _baz$$$ 将发布为 foobar-baz-）。

  1. 在 Python Library 模板部分选择 Create 按钮。

  2. 创建包： 库中任何包含 __init__.py 文件的文件夹都将作为包发布。仓库在初始化时会自带这样一个文件夹——您可以根据需要对其进行重命名或添加额外的包。

  3. 创建模块： 在包文件夹内，您可以添加包含代码的 Python 文件。其他仓库后续将导入这些模块。

  1. 为仓库打标签： 准备好发布版本时，转到“Branches”选项卡，选择“Tags”并创建一个新标签。默认情况下，Python 库仅会在带有标签的提交上发布。要更改此默认行为，您必须修改 build.gradle 文件。

警告

请注意，标签名称必须符合 SLS 版本控制规范，具体请参阅《SLS Versions 文档》↗。

  1. 检查成功完成后，您的库版本即会发布。您可以在仓库的 Tags 列表和 Checks 选项卡中查看检查状态。

默认情况下，仅当您创建标签时，库的更改才会发布。您可以为某个分支的当前状态或特定提交创建标签。检查通过后，您的库将被发布，用户也将能够升级到最新版本。

当发布新版本时，依赖该库的仓库不会自动升级到使用最新版本。 要手动将您的仓库升级到使用最新版本，请参阅有关《发现并使用 Python 库》以及《Conda lock 文件》的文档，以重新解析 Conda 环境。

  1. 确保您为库的消费者赋予了相应权限。默认情况下，用户应在此共享仓库上拥有“Viewer”角色。在 transforms 仓库设置的 Artifacts 选项卡下，添加对该库仓库的引用。

此时，您的库应已可供其他应用和仓库使用。如需了解更多信息，请阅读关于《发现并使用 Python 包》的内容。

查找 Python 库的消费者

要查看哪些仓库正在使用您发布的库，请转到《Artifact Repositories 界面》，并使用 Search 选项卡定位您的库。找到您关注的版本，展开 Downloads 列以查看 </> Consumers 部分。
