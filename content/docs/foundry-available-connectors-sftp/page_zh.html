SFTP

将 Foundry 连接到 SFTP 服务器，以在文件夹和 Foundry 数据集之间同步数据。

支持的功能

功能状态
探索🟢 正式可用
批量导入🟢 正式可用
增量同步🟢 正式可用
导出任务🟡 已弃用（仅限旧版 SFTP 源）
文件导出🟢 正式可用

数据模型

该连接器可将任意类型的文件传输至 Foundry 数据集中。传输过程中及传输完成后，文件格式将保持不变，且不应用任何模式。您可以对输出数据集应用必要的模式，或者编写下游转换来访问数据。

性能与限制

可传输的文件大小没有限制。但是，网络问题可能导致大规模传输失败。特别是，运行时间超过两天的 Foundry 同步将会中断。为避免网络问题，建议使用较小的文件尺寸，并限制每次同步执行时摄入的文件数量。可以将同步任务设为定期调度运行。

设置

  1. 打开《数据连接》应用程序，并在屏幕右上角选择+ 新建源。
  2. 从可用的连接器类型中选择SFTP。
  3. 按照额外的配置提示，使用以下各节中的信息继续设置连接器。

要访问本地部署的 SFTP 服务器，建议使用智能体代理连接。

详细了解如何在 Foundry 中设置连接器。

身份验证

可以使用密码或 SSL 私钥建立与 SFTP 服务器的连接。如果在连接服务器时遇到任何问题，请尝试通过 Foundry 外部的 SFTP 客户端进行连接，以验证身份验证信息是否正确。此连接的用户必须具有对根目录的访问权限，并拥有读取和列出根目录内所有文件及目录的权限。

网络连接

如果您的 SFTP 连接在 Foundry 中运行，您必须添加网络出站策略以将该连接加入白名单。

应针对 SFTP 服务器的主机名（如果通过域名连接）以及该主机名解析出的 IP 地址创建出站策略。正确配置出站策略后，由于主机名验证的原因，非标准连接端口 (22) 可能仍无法工作。如果发生这种情况，请向 Palantir 支持团队报告问题，并提供应用于此连接的策略列表。

如果服务器域名解析到多个域名和/或服务器，则需要将所有关联的域名及其相关 IP 加入白名单。要验证服务器是否解析到多个域名和/或服务器，请从终端对您尝试连接的服务器运行 dig <domain> 命令，并查看应答部分。

在 UNIX 计算机上，通过运行以下命令找到您的服务器域名解析到的 IP 地址：

主机名验证

Foundry 会尝试对所有出站路由进行主机名验证。22 端口之外的网络流量有时未经校验，从而导致连接挂起和/或超时错误。如果即使正确配置了出站策略，错误仍持续出现，请向 Palantir 支持团队报告问题，并附上您希望禁用主机名验证的策略列表。

超时

值为 0 表示连接将无限期等待服务器的所有响应。此超时设置控制连接器与服务器建立连接时的等待时间，以及在服务器上运行每条命令时的等待时间。如果服务器持续响应，则不会触发超时。将此值设置得较低不会阻止大文件传输，但有助于调试挂起的连接。

建立 SFTP 连接时，服务器和客户端必须协商必要的详细信息。这些详细信息包括帮助客户端和服务器决定使用哪些加密算法的信息。由于兼容性问题，您可能需要手动配置这些设置，而不是让它们自动协商。下表列出了可用的配置选项，您可以根据需要调整协商。更多信息，请查阅 常见问题 ↗。

故障排除

借用连接执行 ls 命令失败

如果构建失败并报错 Borrowing connection for ls failed，且错误信息包含 SocketException: Connection reset，请检查构建日志以确认是否已建立与 SFTP 服务器的网络连接。如果日志中不包含 Connection established 和 Remote version string 这两行，说明防火墙阻止了来自连接运行时的出站流量，或者目标 SFTP 服务器不允许入站连接。请检查出站策略和防火墙规则，并联系 SFTP 服务器管理员以解决网络连接问题。以下是成功连接的日志示例：

jschLogMessage: Connecting to <HOSTNAME> port 2232
jschLogMessage: Connection established
jschLogMessage: Remote version string: <value identifying the type of server>

挂起的连接

如果连接挂起且没有任何进展或明显的失败，请将超时时间设置为一个较小的值（例如 1000），以帮助识别是哪个调用挂起了。例如，列出某个特定目录可能花费了更长时间，或者读取文件的某些部分耗费了过长时间。

连接挂起的情况可能是由主机名验证引起的，设置较小的超时时间将允许系统生成失败日志，这有助于支持团队定位连接问题的根本原因。

将数据导出至 SFTP

对于更复杂的场景，请使用专业代码替代方案。

该连接器可以将文件从 Foundry 数据集复制到 SFTP 服务器上的任何位置。

要导出到 SFTP 服务器，首先请为您的 SFTP 连接器启用导出功能。然后，创建新的导出任务。

在许多情况下，Foundry 默认会在数据集的底层文件名前加上 spark/ 前缀。当尝试将该带有前缀的文件名导出到 SFTP 实例时，连接器会默认尝试创建一个 spark 文件夹；如果 Foundry 使用的 SFTP 用户没有在 SFTP 实例上创建文件夹的权限，此操作将会失败。为避免此问题，请在导出前重命名文件。查阅控制输出文件结构文档以获取有关如何在导出前重写文件路径的示例。

在代码中使用 SFTP 数据源

对于更复杂的场景，可以使用专业代码替代方案来连接 SFTP 数据源。

以下示例演示了如何在外部转换中使用 Paramiko ↗ Python 客户端连接到 SFTP 数据源。

将文件写入 SFTP

此示例展示了如何使用 OpenSSH 私钥身份验证将一组 XLSX 文件写入 SFTP 服务器。

1from transforms.api import Output, Input, TransformOutput, TransformInput, transform
2from transforms.external.systems import external_systems, Source, ResolvedSource
3import pandas as pd
4import paramiko
5import io
6
7@external_systems(
8    sftp_source=Source("<source_rid>")
9)
10@transform(
11    output_file=Output(
12        "<output_dataset_rid>"
13    ),
14    xlsx_files=Input("<input_dataset_rid>"),
15)
16def compute(
17    sftp_source: ResolvedSource,
18    output_file: TransformOutput,
19    xlsx_files: TransformInput, # input dataset containing a list of xlsx files
20) -> None:
21
22    # 1. SFTP connection setup
23    hostname = "<hostname>"
24    port = <port> # (usually 22)
25    username = "<username>"
26    private_key_string = sftp_source.get_secret("privateKeyString")
27
28    # 2. Transform private key string to proper input parameter of paramiko
29    # Use StringIO to treat the string as a file-like object
30    private_key_stream = io.StringIO(private_key_string)
31    private_key = paramiko.Ed25519Key.from_private_key(private_key_stream)
32
33    # 3. Define the SFTP client
34    transport = paramiko.Transport((hostname, port))
35    transport.connect(username=username, pkey=private_key)
36    sftp = paramiko.SFTPClient.from_transport(transport)
37
38    # 4. Iterate over input files and upload each to SFTP
39    remote_paths = []
40    fs = xlsx_files.filesystem()
41    input_files = fs.ls()
42    for f in input_files:
43        with fs.open(f.path, "rb") as fileobj:  # Open in binary mode
44            remote_path = get_remote_path(f.path)
45            with sftp.file(remote_path, "wb") as remote_file:
46                remote_file.write(fileobj.read())  # Write bytes directly
47            remote_paths.append({"file_name": f.path, "remote_path": remote_path}) # record mapping of file name to uploaded path
48
49    # 3. Return dataset of remote paths
50    output_file.write_pandas(pd.DataFrame.from_records(remote_path))
51
52
53def get_remote_path(file_path):
54    """
55    Define the remote path where the file will be uploaded.
56    """
57    remote_target_folder = "<example_target_folder>"
58    clean_file_path = file_path.replace(" ", "_")
59    return f"{remote_target_folder}/{clean_file_path}"

旧版 SFTP 连接器

以下章节介绍旧版 SFTP 连接器，仅在维护该连接器的现有使用场景时供参考。任何新建 SFTP 数据源的操作，请遵循前述章节中的步骤。

迁移至新版 SFTP 连接器

从旧版 SFTP 连接器迁移至新版连接器的过程无法自动化，必须手动完成。新版连接器支持几乎完全相同的配置选项，因此需要新建一个数据源来替换原有数据源。

请注意：

  • 无法从旧数据源中提取密码，如果已遗失相关访问权限，须向服务器管理员申请获取。
  • 不再支持 privateKeyFile 和 privateKeyPassphrase 参数。若使用私钥认证，请将私钥内容直接填入数据源设置中。
  • 不再支持 knownHostsFile 参数，必须在数据源设置中正确配置 base64HostKey。

如需配置旧版 SFTP 连接器，请进入“数据连接”应用，然后选择 + 新建数据源。在数据源选择界面中，向下滚动至 高级 选项，接着选择 自定义数据源。请根据配置界面的提示继续完成 SFTP 连接器的设置。

使用旧版 SFTP 连接器导出

通常不建议使用导出任务将数据写回外部源，以下文档仅为支持现有的旧版导出任务而保留。任何新的导出操作必须使用更新后的一等公民SFTP 连接器来创建。

要导出数据，必须配置导出任务。导航至包含要导出目标连接器的项目文件夹。右键单击连接器名称，然后选择创建数据连接任务。

在数据连接视图的左侧面板中：

  1. 验证Source名称与要使用的连接器是否匹配。
  2. 添加名为inputDataset的Input。输入数据集即正在导出的 Foundry 数据集。
  3. 添加名为outputDataset的Output。输出数据集用于运行、调度和监控任务。
  4. 最后，在文本框中添加 YAML 块以定义任务配置。

左侧面板中显示的连接器和输入数据集标签并不反映 YAML 中定义的名称。

标志文件

对于给定的构建，一旦所有数据复制完毕，连接器即可向 SFTP 服务器写入一个空的标志文件。这个空文件表示内容已准备好供消费，并且不会再进行修改。标志文件将写入directoryPath。但是，如果启用了createTransactionFolders，则每个写入内容的文件夹都将生成一个标志文件。如果启用了标志文件，并且标志文件名为confirmation.txt，则在构建中导出的文件写入完成后，所有标志文件将一次性写入。

标志文件在构建结束时写入，而不是在子文件夹导出时写入。

如果 SFTP 服务器上的文件比标志文件新，通常表明之前的导出未成功，或者该文件夹的导出正在进行中。

扩展协商

如果客户端和服务器都支持 SFTP 协议的高级特性，SFTP 协议允许客户端使用这些高级功能。对于大多数行业标准的 SFTP 实现，我们建议保持启用扩展。然而，如果存在兼容性问题，扩展可能会导致连接失败。Foundry SFTP 连接器允许根据需要禁用扩展。通常，最佳实践是仅禁用必要的少量扩展，而不是禁用所有扩展。
