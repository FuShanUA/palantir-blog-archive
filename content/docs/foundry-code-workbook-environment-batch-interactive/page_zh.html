批量构建与交互式构建

通过工作簿界面运行作业，以交互方式使用 Code Workbook 时，所有作业都会使用与该工作簿关联的 Spark 模块。如下图所示，你可以在任意会话的会话历史对话框中查看 Spark 模块 ID。

由于在同一项目中使用相同环境的各个工作簿，会为每位用户分配同一个 Spark 模块，因此你的交互式作业可能会排队等待其他工作簿的作业完成。例如，同一个模块最多可以同时运行五个 Python 作业。第六个作业将显示为“在 Code Workbook 中排队”。

在批量构建中（例如计划构建或通过数据集预览发起的构建），一次构建会在构建过程中为每个环境使用一个 Spark 模块。例如，如果计划构建包含在多个 Code Workbook 中创建的数据集，且这些工作簿都使用相同的环境，那么该构建将对所有作业使用同一个 Spark 模块。交互式作业不会被路由到该 Spark 模块。

当所需输出已保存为数据集，且你无需逐一运行转换来迭代代码时，我们建议使用批量构建。这适用于以下情况：

  • 你希望在同一个工作簿或使用相同交互式 Spark 模块的其他工作簿中，运行一些耗时的转换。
  • 你希望数据集按一定周期更新。例如，若希望在出现新输入数据时或每天上午 9 点查看更新后的数据，应使用计划批量构建。

在工作簿中，使用打开数据集在数据集预览中查看数据集，并点击屏幕右上角的构建按钮进行构建。若要构建多个数据集，请导航至页面顶部的齿轮图标，选择探索数据血缘关系。接着，选中需要构建的数据集，并在右侧边栏中选择构建它们。

或者，使用相同的侧边栏并点击日历图标，设置定期计划来构建数据集。了解更多关于设置计划的信息。

要确认作业是在批量构建还是交互模式下构建，请导航至构建应用程序并点击“详情”按钮。

详情将列出 Spark 模块的 ID，以及 isInteractive 是否为 true。若为 true，则作业正以交互模式运行。若为 false，则作业正以批量构建模式运行，且不与任何交互式作业共享 Spark 模块。
