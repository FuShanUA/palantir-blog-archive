SAP ERP

The SAP ERP connector allows you to connect Foundry to SAP's on-premise ERP Central Component (ECC) and S/4 HANA (on-premise and Cloud Private Edition ↗). The SAP ERP connector enables Foundry to interact with various types of SAP data, including:

  • SAP Application Tables
  • SAP CDS Views
  • SAP HANA Information Views
  • SAP BW Content Extractors (remote connections only)
  • SAP Transaction Codes and ABAP Reports
  • SAP Function Modules
  • SAP Media (DMS documents)

Using the SAP ERP source requires the installation of the Palantir Foundry Connector 2.0 for SAP Applications add-on on the target SAP application layer.

Setup

  1. Open the Data Connection application and select + New Source in the upper right corner of the screen.
  2. Select SAP ERP from the available connector types.
  3. Follow the additional configuration prompts to continue the setup of your connector using the information in the sections below.

Learn more about setting up a connector in Foundry.

Connection type

The SAP ERP connector supports two connection types:

Connection typeDescription
DirectConnects directly to the SAP system. This is the default.
Remote (via Gateway)Connects to a remote SAP system through a gateway. Enable the Connect via Gateway toggle and provide a Context value to identify the remote system.

The connection type determines which SAP object types are available. Learn more about remote connections and remote agent configuration.

Authentication

The SAP ERP connector supports the following authentication methods:

Authentication methodDescription
Basic AuthProvide the username and password of the technical user created when installing the Connector.
Authentication tokenProvide a token to authenticate.
Custom authentication headerProvide a custom authentication header.
No authenticationUse this option if authentication is set up on the agent machine via certificates.

Networking and connectivity

Make sure to properly configure egress policies to allow Foundry to reach the SAP system. For on-premises SAP environments, agent proxy policies are typically required to route traffic correctly.

Many SAP systems use custom-signed certificates, which can cause SSL handshake exceptions when configuring the connection for the first time. Make sure you have the correct custom certificates from your system and add them to the source.

SAP Data Accelerator

To reach this SAP system through SAP Data Accelerator (SAP's managed connectivity service for systems exposed through an SAP Cloud Connector), set the source's host and port to your Cloud Connector virtual host and port. Then, assign the client certificate provided by Palantir, allow egress to the SAP Data Accelerator control plane and data plane, and add the SAP Data Accelerator property with your control plane URL. This path requires using a Foundry worker.

增量同步

增量同步支持从同一表进行追加式事务。要启用增量同步，请将 事务类型 设置为 APPEND。

可用的增量模式取决于对象类型：

增量模式ERP 表CDS 视图HANA 视图BW 内容提取器
多字段✓✓✓
拼接字段✓✓✓
变更凭证表✓
孪生表✓
SAP 内置增量✓
  • 多字段：导入指定字段中任意一个大于或等于已导入最大值的行。字段之间用逗号分隔。
  • 拼接字段：与多字段类似，但会将字段值拼接在一起，而不是通过 OR 进行组合。
  • 变更凭证表：根据 SAP 变更凭证表中的更新来导入行。提供两种子模式：仅 CDPOS 或 CDHDR 和 CDPOS。
  • 孪生表：当独立“孪生”表中的某个字段满足增量条件时，导入目标表中的行。
  • SAP 内置增量：使用提取器原生的变更数据捕获机制。

增量字段最好是单调递增的值。系统采用“大于或等于”比较方式，以避免在同步运行至某日期中途时丢失数据。Foundry 数据集中可能会出现重复值，应在转换管道的第一步中予以清除。

事务码和功能模块对象类型不支持增量同步，它们始终以快照方式运行。

对于表类型的增量同步，每次同步最大行数 设置限制了每次运行返回的大致行数。如果间歇性问题（如网络故障）中断了长时间运行的同步，可使用该设置将大表的初始同步拆分为一系列更小、更具弹性的运行任务。

孪生表配置

当使用 孪生表 作为增量模式时，增量孪生表 设置用于指定孪生表的名称，而 增量孪生映射 设置用于定义主表与孪生表之间的连接条件。映射条目采用以下格式：

{PRIMARY_TABLE_NAME}-{FIELD_NAME}={TWIN_TABLE_NAME}-{FIELD_NAME}

多个连接条件之间用分号联合（相当于 AND 运算符）。

重置增量状态

要强制进行全量重新加载并重新初始化增量摄取，请在同步中启用 重置增量状态 开关。这将执行来自 SAP 的数据全量快照，并替换数据集中的所有文件。同步完成后，请关闭该开关，以便后续同步恢复增量追加。

ERP 表、CDS 视图、HANA 视图和 BW 内容提取器对象类型支持重置增量状态。

Filters

Add condition groups, then configure field, operator, and value rows within each group. All field names must match the SAP data dictionary.

  • For date columns, use the format YYYYMMDD (for example, 20210101 for January 1, 2021).
  • The is like and is not like operators support the * wildcard (for example, A*12* matches any string starting with A and containing 12).
  • Filter values can also use dynamic filter keywords and date calculation functions.

Dynamic filters

Dynamic filter values use special keywords and date calculation functions for more flexible filter expressions. Available from add-on version SP26 and later. Use them anywhere a filter value is accepted.

Webhook

示例：通过 BAPI 回写至 SAP（转换）


以下《外部转换》会对输入数据集的每一行调用一次 BAPI_SALESORDER_CHANGE，以更新各销售文档的采购日期，并将 SAP 的 RETURN 表记录至输出数据集中供审查。



```python
from transforms.api import Input, Output, transform
from transforms.external.systems import external_systems, Source, ResolvedSource


@external_systems(
    sap_source=Source(""),
)
@transform(
    sales_order_updates=Input(""),  # 列: sales_doc_id, purchase_date (YYYYMMDD)
    writeback_results=Output(""),
)
def update_purchase_dates(ctx, sap_source: ResolvedSource, sales_order_updates, writeback_results):
    base_url = sap_source.get_https_connection().url
    client = sap_source.get_https_connection().get_client()

    results = []
    for row in sales_order_updates.dataframe().collect():
        response = client.post(
            f"{base_url}/v2/function/BAPI_SALESORDER_CHANGE/writeback",
            json={
                "commit": True,
                "input": {
                    "SALESDOCUMENT": row.sales_doc_id,
                    "ORDER_HEADER_IN": {"PURCH_DATE": row.purchase_date},
                    "ORDER_HEADER_INX": {"UPDATEFLAG": "U", "PURCH_DATE": "X"},
                },
                "output": ["RETURN"],
            },
        )
        response.raise_for_status()
        results.append({
            "sales_doc_id": row.sales_doc_id,
            "purchase_date": row.purchase_date,
            "status_code": response.status_code,
            "sap_return": str(response.json().get("RETURN", [])),
        })

    writeback_results.write_dataframe(ctx.spark_session.createDataFrame(results))
```


对于需要 commit 语句来持久化更改的标准 BAPI，请将 "commit" 设置为 true。在调用函数之前，请使用 metadata 端点（GET /v2/function/{functionName}/metadata）来探查该函数预期的输入和输出结构。

2from transforms.api import transform
3from transforms.external.systems import external_systems, Source, ResolvedSource
4from transforms.mediasets import MediaSetOutput
5
6
7@external_systems(
8    sap_source=Source("<source_rid>"),
9)
10@transform(
11    documents=MediaSetOutput("<media_set_rid>"),
12)
13def fetch_dms_documents(ctx, sap_source: ResolvedSource, documents):
14    base_url = sap_source.get_https_connection().url
15    client = sap_source.get_https_connection().get_client()
16
17    # 列出 DMS 中的文档，按文档类型和文件格式进行过滤。
18    list_response = client.get(
19        f"{base_url}/v2/get_documents",
20        params={"DOCUMENTTYPE": "CPT", "FILEFORMAT": "pdf"},
21    )
22    list_response.raise_for_status()
23
24    for doc in list_response.json():
25        document_id = doc["DOCUMENT_ID"]
26        document_name = doc["DOCUMENT_NAME"]
27
28        download_response = client.get(
29            f"{base_url}/v2/get_document",
30            params={"DOCUMENT_ID": document_id},
31        )
32        download_response.raise_for_status()
33
34        documents.put_media_item(io.BytesIO(download_response.content), document_name)



例如，要驱动从 Foundry 输入数据集的数据抽取，仅获取由一组 EBELN（采购订单）值引用的文档，可以通过 OBJECTLINK 过滤器调用 /v2/get_documents，或者直接将输入行中的每个 DOCUMENT_ID 传递给 /v2/get_document。
