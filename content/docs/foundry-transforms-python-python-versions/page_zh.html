Python 版本支持

随着 Python 开发的不断推进，Python 软件基金会将旧版 Python 标记为“生命周期结束”（EOL） ↗；超过 EOL 日期的版本不再受官方支持。

Foundry 与 Python 版本管理

从 Python 3.9 版本起，Foundry 将遵循 Python 软件基金会的 EOL 时间表，不再支持已废弃的 Python 版本。已废弃的 Python 版本会给平台带来安全风险，且可能与平台运行所需的技术不兼容（例如，Apache Spark 不支持旧版 Python）。Palantir 会通知正在使用即将到达 EOL 的 Python 版本的 Foundry 用户。

如何判断我的工作流是否正在使用已废弃的 Python 版本？

如果您的某个工作流使用了已废弃的 Python 版本，系统会通过弹窗和横幅通知您。这些通知将包含如何从已废弃版本升级的说明。

哪些 Python 版本是安全的？

Python 软件基金会的版本生命周期结束表 ↗会持续更新，指明了哪些 Python 版本是安全的。我们建议您始终使用最新受支持的版本。您可以参考下表，了解 Foundry 对 Python 版本的支持情况。

警告

Palantir 不对运行于超过 EOL 的 Python 版本上的工作流提供支持。

Python 版本废弃

平台将创建相关活动，帮助用户在生命周期结束之初识别受影响的工作流。

自 2026-10-31 起，Foundry 将不再支持 Python 3.10。我们将通知用户如何识别任何受影响的工作流，并在截止日期前迁移至未废弃的 Python 版本。

自 2026-01-31 起，Foundry 不再支持 Python 3.9。用户应迁移至未废弃的 Python 版本。

自 2025-01-31 起，Foundry 不再支持 Python 3.6、3.7 和 3.8。用户应迁移至未废弃的 Python 版本。

请参阅《环境故障排查指南》，详细了解如何更改您的 Python 版本。
