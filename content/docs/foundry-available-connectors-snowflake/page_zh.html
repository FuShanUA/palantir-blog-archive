设置

  1. 打开数据连接应用，在屏幕右上角选择+ 新建源。

要确认需加入允许列表的 Snowflake 平台账户主机名和端口号，您可以在 Snowflake 平台控制台中运行以下命令。请确保至少将 SNOWFLAKE_DEPLOYMENT 和 STAGE 的条目作为出口策略添加到 Foundry 中。

从 Foundry 到 Snowflake 平台的连接通常来自您环境的默认公共网关 IP。但是，同一云提供商内的流量（例如 AWS-AWS 或 Azure-Azure）可能使用不同的路由，并需要通过 PrivateLink 建立连接。有关各云提供商所需的额外设置，请参阅下文，或联系您的 Palantir 代表获取更多指导。

托管在 S3 上的 Snowflake 平台实例

如果您的 Snowflake 平台实例配置为通过 VPCE ↗ 路由内部 S3 暂存区流量，则必须手动配置 Snowflake JDBC 驱动程序，使其不使用自定义 VPCE 域。否则，驱动程序将被路由到自定义 VPCE 域（从 Foundry 的 VPC 无法访问该域），并且将无法连接到格式为 <bucketname>.bucket.vpce-<vpceid>.s3.<region>.vpce.amazonaws.com 的 URL。

您可以通过在实例的连接详情中添加 JDBC 连接属性来手动配置此项，其键为 S3_STAGE_VPCE_DNS_NAME，值字段为空（等同于将其设置为 null）。 随后，S3 暂存区流量将通过 AWS S3 网关端点（<bucketname>.bucket.s3.<region>.vpce.amazonaws.com）进行路由，该端点可保持私有连接，从而确保流量不会通过公共互联网进行路由。

查阅我们的 PrivateLink 出口文档 ↗ 以了解更多信息。

对于依赖与您的 Foundry 实例位于同一区域的 S3 存储桶的出口策略，请确保您已按照 Amazon S3 存储桶策略文档中针对受影响存储桶的详细说明，完成了额外的配置步骤。

托管在 Azure 上的 Snowflake 平台实例

Foundry Snowflake 连接器所使用的 Snowflake JDBC 驱动程序在获取数据时，可能会尝试直接连接到底层的“内部暂存区”存储桶。对于托管在 Azure 上的 Snowflake，由于 Azure 托管的 Foundry 注册项通过 Azure 服务端点路由流量，必须按照以下说明显式地将 Foundry 到底层暂存区存储桶的网络连接加入允许列表。

收集有关您的 Snowflake 仓库的所需信息

您需要提供有关 Azure 托管的 Snowflake 仓库的以下信息，以便与 Foundry 建立网络连接：

  • 完整的系统允许列表域名。
  • 您的Azure 存储帐户标识符，从 Snowflake 获取。
系统允许列表域名

使用 SYSTEM$ALLOWLIST 命令获取可能需要连接成功的完整域名列表。

  • 注意：这与上面用于定义出站策略的命令相同，并在数据连接的网络面板提示中进行了说明。
  • 此列表将包含用作您的 Snowflake 仓库暂存区的 Azure 存储桶的域名。
Azure 存储帐户标识符

对于从 SYSTEM$ALLOWLIST 命令返回的 Azure 存储桶，您还需要检索存储帐户标识符。

  • 如果您使用的是 Snowflake 标准版或企业版，您需要向 Snowflake 支持团队提交工单以请求存储帐户标识符。
  • 如果您使用的是 Snowflake 商业关键版 ↗，可以通过以下步骤检索存储帐户标识符：
    • 将帐户的 ENABLE_INTERNAL_STAGES_PRIVATELINK ↗ 参数设置为 TRUE。
    • 然后，调用 SYSTEM$GET_PRIVATELINK_CONFIG() ↗ 函数，该函数会返回一个名为 privatelink-internal-stage 的字段，其中包含 Azure 存储帐户资源标识符。
      • 请注意，即使您没有通过 PrivateLink 进行连接，您仍然需要检索并提供存储帐户资源标识符。

完整的 Azure 存储帐户资源标识符将采用以下格式：

/subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}/providers/Microsoft.Storage/storageAccounts/{storageAccountName}

有关如何直接在 Azure 控制台中查找 Azure 存储帐户资源 ID 的更多信息，请参阅 Azure 文档 ↗。使用 VNET 规则和存储帐户标识符限制跨帐户网络流量符合 微软发布的最佳实践 ↗，并且应应用于从 Azure 计算实例内连接到 Azure 托管的 Snowflake 仓库的所有连接。

允许从 Foundry 到与您的 Snowflake 仓库关联的 Azure 存储帐户的出站流量

现在您已经收集了有关 Snowflake 仓库的所需信息，可以创建使 Foundry 能够访问您的 Snowflake 数据所需的策略。

  1. 为 Azure 存储内部暂存区创建标准出站策略，并将其附加到您的 Snowflake 源。

    • 请注意，您应该为 SYSTEM$ALLOWLIST 命令返回的所有内容添加策略，而不仅仅是存储桶域名。

  2. 创建Azure 存储策略，并粘贴存储帐户资源标识符。

导航回数据连接中的 Snowflake 源，确认您可以浏览该源并运行同步。

Iceberg 表（仅限虚拟表）

本文档的虚拟表部分详细介绍了如何集成在 Snowflake Horizon Catalog 中注册的 Iceberg 表。此功能需要与存储 Iceberg 表的外部卷建立网络连接。您必须为每个外部卷创建网络出站策略，因为 Foundry 是直接读取和写入存储位置，而不是使用 Snowflake 计算仓库查询表。

有关外部卷以及如何确定表存储位置的更多信息，请参阅 Snowflake 官方文档 ↗。

为外部卷配置出口策略后，网络流量即可从 Foundry 流向该存储位置。不同云服务提供商的网络控制机制可能有所不同，因此需确保存储位置上的所有网络控制均允许来自 Foundry 的网络流量。了解有关识别 Foundry 流量源 IP 地址的更多信息。.

此外，请查阅此《网络》文档的其他章节，确保正确建立与同 Foundry 实例处于同一云区域的外部卷的连接。

虚拟表

本节提供了将虚拟表与 Snowflake 平台数据源结合使用的更多细节。本节内容不适用于同步至 Foundry 数据集的场景。

如今，使用虚拟表访问 Horizon Catalog 中注册的 Iceberg 表时，Snowflake 平台连接器提供了增强功能。Foundry 利用 Horizon Catalog 暴露的 Iceberg REST API 访问表，并直接在底层存储位置读写数据，这些底层存储位置在 Snowflake 平台中均配置为外部卷。此外，Foundry 采用 Horizon Catalog 凭证派发机制，确保对云对象存储的安全访问。这还能提升针对这些表的读写性能。

满足以下条件时，连接器会自动启用 Iceberg 功能：

  1. 配置网络出口策略，允许 Foundry 连接至存储该表的外部卷。
  2. 在数据源上配置凭证，且该凭证具有从 Horizon Catalog 获取派发凭证的权限。



Foundry 使用 Iceberg 客户端建立连接，直接向存储位置读取或写入表，无需占用 Snowflake 平台的计算资源。然而，在某些元数据查询（例如判断所访问表的类型）中，Foundry 仍会使用数据源上配置的仓库。

有关通过 Snowflake Horizon Catalog 使用外部引擎查询 Iceberg 表的更多信息，请参阅《Snowflake 官方文档 ↗》。向 Horizon Catalog 执行写入操作目前是 Snowflake 平台的一项私有预览功能。有关启用外部卷网络访问的详细信息，请参阅本文档的《网络》章节。

若未能满足上述任何要求，Foundry 会将 Iceberg 表视同常规 Snowflake 表处理。此时与 Snowflake 平台的连接机制与其他 Snowflake 数据（如表、视图或物化视图）保持一致，并依赖 Snowflake 平台计算仓库来完成表的读写操作。

数据源配置要求

在使用虚拟表时，请注意以下数据源配置要求：

  • 必须使用 Foundry worker 数据源。虚拟表不支持使用 agent worker 连接。
  • 确保已按照本文档的网络配置部分的要求，建立双向网络连通性并完成白名单配置。
  • 如果在代码仓库中使用虚拟表，请参阅《虚拟表文档》，了解所需附加数据源配置的详细信息。
  • 必须在连接详情中指定仓库。
  • 提供的凭证必须具备该仓库的使用权限。

有关更多详细信息，请参阅上方的《连接详情》部分。

计算下推

使用虚拟表时，Foundry 支持将计算下推至 Snowflake 平台。虚拟表输入利用了 Snowflake Spark 连接器 ↗，该连接器内置了对谓词下推的支持。

当使用注册到同一数据源的 Snowflake 平台虚拟表作为管道的输入和输出时，可以将计算完全联邦化至 Snowflake 平台。若要将计算下推至 Snowflake 平台，请查阅 《Python 文档》 了解详细信息。若要在管道构建器中将计算下推至 Snowflake 平台，请查阅 《外部管道文档》。

在代码中使用 Snowflake 数据源

对于更复杂的场景，您可以采用代码方式连接到 Snowflake 数据源。

本示例使用基于 OIDC 的 OAuth 凭证从 Snowflake 表中读取数据。

1import polars as pl
2from transforms.api import transform, Output, LightweightOutput
3from transforms.external.systems import ResolvedSource, external_systems, Source
4import snowflake.connector
5
6
7@external_systems(
8    snowflake_source=Source("<source_rid>")
9)
10@transform.using(
11    output=Output("<output_dataset_rid>")
12)
13def read_from_snowflake(output: LightweightOutput, snowflake_source: ResolvedSource):
14    conn = snowflake.connector.connect(
15        authenticator="oauth",
16        token=snowflake_source.get_session_credentials().get().access_token,
17        account="<account_identifier>",
18        warehouse="<warehouse_name>",
19        database="<database_name>",
20    )
21
22    cursor = conn.cursor()
23    try:
24        cursor.execute("SELECT * FROM my_table LIMIT 100")
25        columns = [col[0] for col in cursor.description]
26        rows = cursor.fetchall()
27    finally:
28        cursor.close()
29        conn.close()
30
31    df = pl.DataFrame(rows, schema=columns)
32    output.write_table(df)

22

数据模型

请注意，Foundry 会将 array ↗、object ↗ 和 variant ↗ 类型的列解析为 string 类型。这是由于数据源的类型可变性所致。

例如，Snowflake 平台 中的数组 [ 1, 2, 3 ] 会被 Foundry 解释为字符串 "[1,2,3]"。

详情请参阅 Snowflake 平台 的官方文档 ↗。
