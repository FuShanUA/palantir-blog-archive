发现并使用 Python 库

代码仓库允许你导入并使用公共库以及 Foundry 生成的库。以下信息仅适用于 Python 库。

发现 Python 库

要搜索 Python 库，请点击代码仓库环境左侧面板上的“包”选项卡。使用搜索框搜索可用的库。

点击库名以展开库详情和 添加库 选项，该操作会将推荐版本的库添加到你的分支中。

添加库后，Code Assist 依赖项将刷新，随后你可以从该库中可用的包导入模块。

添加异常依赖项

添加 Python 库有时需要将关键制品仓库添加到你的代码仓库中，并在项目中进行引用。在这种情况下，系统会弹出一个对话框，要求你确认此操作。

依赖项对话框会高亮显示你无权访问的任何必需仓库。你需要拥有这些仓库的访问权限才能使用该库。

代码仓库会尝试为你的库寻找可用的支撑仓库并自动使用它们。如果你需要特定的支撑仓库，可以直接在仓库的《制品设置》页面中进行设置。

锁定特定库版本

如果你需要特定版本，可以点击设置按钮并从列表中选择所需版本来对其进行锁定。这允许你设置不同的版本来运行转换和测试。

请注意，锁定库的特定版本后，该版本将被添加到你的 meta.yaml 文件中，但不会通过 Task Runner 进行安装。请验证检查是否成功通过，然后重启 Code Assist 以将更改应用到你的 meta.yaml 文件。随后，你就可以在仓库中开始使用所选版本的库了。

警告

锁定特定版本时请务必谨慎。锁定版本可能会阻止你的代码获取重要更新。请始终确保审查并更新你的依赖项。

审查库更改

从仓库中添加和移除 Python 库的操作与任何其他代码更改的行为一致。你应该提交更改并将它们合并到受保护分支中。在拉取请求中，库更改将显示为 meta.yaml 文件中的更改。

在创建拉取请求时，你偶尔会在 .lock 类型的文件上看到合并冲突。当你在分支上工作时发生了更新，就会出现这种情况。在这种情况下，请接受双方的更改并继续处理你的拉取请求。

使用发布到共享通道的库 [已弃用]

警告

这是一项已弃用的功能，可能你的环境中不可用。

如果你的库已发布到共享通道（参见《发布 Python 库》），则必须编辑 Python 子项目的 build.gradle。你将要修改的是隐藏文件，因此在继续之前，请确保选择了“显示隐藏文件”。你的 Python 转换项目需要有权访问发布共享库的通道。在 Python 转换子项目文件夹的 build.gradle 文件中，添加以下内容：



已复制！
1 transformsPython {
2     sharedChannels  "libs"
3}

你必须编辑 Python 子项目文件夹中的 build.gradle 文件，而不是仓库根目录下的那个文件。请务必将其添加到文件末尾，以确保 apply plugin: 已提前执行并能正确处理它。

制品仓库设置

通常情况下，在处理 Python 库时无需访问仓库设置。所需的制品仓库会在你添加 Python 包时自动添加。应避免直接在设置标签页中编辑已引用的 Python 仓库列表。

在使用共享库时，对相关仓库的引用会被添加到项目中。你可以通过访问仓库设置标签页的“制品”部分来查看已引用的仓库列表。

如果你是手动添加共享库依赖项，而不是通过制品包搜索功能添加的，则必须通过制品设置将该库的仓库添加到你的使用仓库的后端仓库中。请注意，不建议采用这种方式，因为包搜索功能会自动为你完成此操作。

任务运行器

任务运行器仅在较新的仓库中可用。你可以通过显示隐藏文件并查看 templateConfig.json 文件来检查仓库版本。

  • 如果你的仓库的 parentTemplateId 为 transforms，请确保 parentTemplateVersion 为 8.220.0 或更高版本，并且 Python 子模板为 1.484.0 或更高版本。
  • 如果你的仓库的 parentTemplateId 为 python-library，请确保 parentTemplateVersion 为 1.497.0 或更高版本。

你可以升级你的仓库来启用此功能。

启用任务运行器后，从左侧面板的库标签页添加包时，将不再提供新的 Code Assist 工作空间，而是会直接在当前环境之上开始安装请求的包，同时回传底层过程的所有日志。

如果安装成功，锁定文件将随新环境一同更新。如果安装失败，任务运行器底部面板会显示错误信息，可用于进一步排查问题。

任务运行器仅会更新 run 环境，目前不支持安装仅用于测试的依赖项。

meta.yaml 文件

警告

应避免编辑 meta.yaml 文件，因为这极易引发错误。建议优先通过库搜索界面来添加库。

要在代码仓库中使用 Python 库，必须将其包含在 conda_recipe/meta.yaml 文件中。通过仓库库搜索界面添加库时，系统会自动执行此操作。



Copied!
1requirements:
2...
3run:
4    - python
5    ...
6    - {LIBRARY NAME} # Replace this with the name of your shared library.

添加库后，点击 meta.yaml 文件顶部的“刷新依赖项”。这将确保代码辅助功能更新新依赖项，以便你继续从可用包中导入模块。

有关 meta.yaml 文件的更多信息，请参见相关文档。

Python 包的 Conda 解析

Conda 是一款开源的、与语言无关的包和环境管理器，在代码仓库中用于解析包依赖关系，并将包集合安装到独立的环境中。欲了解更多信息，请查阅官方 Conda 文档或《环境创建简介》。

Conda 锁定文件

在代码仓库中运行检查时，我们会为 meta.yaml 文件中列出的包列表解析 Conda 环境，并生成扩展名为 .lock 的隐藏 Conda 锁定文件；这些锁定文件用于保存 Conda 环境。这种预先解析好的环境使得你提交代码时的后续检查能够更快地完成 Conda 解析。

在以下情况下，我们将重新解析 Conda 环境并写入新的锁定文件：

  • meta.yaml 文件中的包列表发生了更改。
  • 仓库已升级到较新的模板版本。
  • 在你的锁定文件中发现了被召回的包。
  • 隐藏的 Conda 锁定文件已被删除或编辑。

当我们重新解析环境并写入新的锁定文件时，初始提交哈希将被标记为 SUPERSEDED，随后跟着另一个用于重写锁定文件的提交哈希。在需要重新解析环境时，检查可能需要更长时间才能运行完成。

在平台外下载已发布的 Python 库

你可以从 Foundry 外部下载在 Palantir Foundry 中发布的库：

  1. 获取用户令牌。在 Foundry 中，导航至你的用户账户设置并选择令牌。选择创建令牌，输入令牌的名称和描述，然后选择生成。复制你的令牌。
警告

请勿与其他应用程序或用户分享你的令牌；恶意行为者可能会利用它来冒充你。

  1. 查找你的 Python 库的 <identifier>。导航至目标 Python 库的代码仓库。浏览器 URL 现在应具有类似以下的形式：https://<my-foundry-url>/workspace/data-integration/code/repos/<identifier>/contents/refs%2Fheads%2F<branch>。在 URL 中找到 <identifier>，其形式类似于 ri.stemma.main.repository.XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX。
  2. 获取库的包索引。在本地计算机上打开终端并运行以下命令： curl -H "Authorization: Bearer $TOKEN" https://$STACK_URL/artifacts/api/repositories/$IDENTIFIER/contents/release/conda/$PLATFORM/repodata.json 其中 $TOKEN 是您的用户令牌，$STACK_URL 与上一步中的 <stack-url> 相同，$IDENTIFIER 是上一步中的 <identifier>，而 $PLATFORM 是仓库平台（例如 noarch、linux-64）。上述命令的输出是库内容的索引（即它所包含的包）。每个包的格式为 <name>-<version>-<build-string>.tar.bz2。
  3. 下载特定包。您可以在终端中运行以下命令，从 Python 库下载特定的包： curl -H "Authorization: Bearer $TOKEN" https://$STACK_URL/artifacts/api/repositories/$IDENTIFIER/contents/release/conda/$PLATFORM/$PACKAGE 其中 $PLATFORM 是包平台（例如 noarch、linux-64），而 $PACKAGE 的格式为 <name>-<version>-<build-string>.tar.bz2。

在任务运行器中运行任务

您可以通过手动指定命令，从任务运行器中手动触发包安装。

例如，要安装 pandas，请在底部面板中选择 任务运行器 选项卡，然后输入以下命令：

install --packageSpecs=pandas

任务运行器还支持以下命令：

  • uninstall：从运行环境中卸载包。
    • 注意：此命令还将卸载任何依赖于指定包的包。此命令仅在较新的仓库中可用。如果您的仓库具有 transforms 的 parentTemplateId，请确保 Python 子模板处于 1.525.0 或更高版本。类似地，如果您的仓库具有 python-library 的 parentTemplateId，请确保 parentTemplateVersion 处于 1.525.0 或更高版本。
    • 用法：对于 Conda 包，使用 uninstall --packageSpecs=<eg.numpy>。
  • formatCode：格式化仓库中的文件。使用 Black 格式化代码，并使用 pyproject.toml 进行自定义格式化程序配置。有关更多信息，请参阅 Black 格式化程序配置文件文档 ↗。
  • whoneeds：显示当前运行环境中需要安装某个包的包树。
    • 注意：此命令仅在较新的仓库中可用。如果您的仓库具有 transforms 的 parentTemplateId，请确保 Python 子模板处于 1.522.0 或更高版本。类似地，如果您的仓库具有 python-library 的 parentTemplateId，请确保 parentTemplateVersion 处于 1.522.0 或更高版本。
    • 用法：whoneeds --packageSpec=<eg.transforms>
  • tasks：列出所有可运行的任务。
