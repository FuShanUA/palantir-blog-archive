使用 Velox 加速 Spark

Spark 加速是一项利用底层硬件优化来提升 Spark 作业性能的技术。通过运用特定平台的功能，原生加速旨在大幅缩短处理大规模数据工作负载所需的时间，从而加快作业执行速度并提升资源利用率。

Velox ↗ 是一个可复用、高性能、底层数据处理库，提供了一系列用于构建高性能数据处理系统的基础组件。它旨在作为构建上层数据处理系统的基础，并在 Foundry 中用于加速 Spark 作业。

阅读有关 Foundry 中原生加速的更多内容。

快速开始

Spark 加速可用于任何现有的 Spark 管道。无需对业务逻辑做任何修改。

要在 Python 转换管管中使用原生加速，必须完成以下步骤：

  1. 将 Python 代码库升级至最新版本。
  2. 配置堆外内存配置文件。
  3. 启用 VELOX 后端，如下方代码片段所示：
```python from transforms.api import configure, ComputeBackend, Input, Output, transform_df @configure( ["EXECUTOR_MEMORY_MEDIUM", "EXECUTOR_MEMORY_OFFHEAP_FRACTION_HIGH"], backend=ComputeBackend.VELOX) @transform_df( Output('/Project/folder/output'), source_df=Input('/Project/folder/input'), ) def compute(source_df): ... ```

为加速的 Spark 配置内存

要优化原生加速的 Spark 项目，请先为堆外内存使用 EXECUTOR_MEMORY_OFFHEAP_FRACTION_HIGH 设置。这部分内存由 Velox 使用，用于在 JVM 之外处理部分任务。观察性能表现，并根据需要上下调整堆外内存大小。

若要使用按比例分配的堆外内存配置，必须同时设置 EXECUTOR_MEMORY_X 配置项。你的任务通常已经具备该项配置。
