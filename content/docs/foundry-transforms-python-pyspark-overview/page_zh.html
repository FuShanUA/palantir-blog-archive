PySpark 转换

PySpark 是一种包装语言，允许你与 Apache Spark 后端交互以快速处理数据。Spark 能够在分布式的服务器网络中处理超大规模数据集，合理使用能带来显著的性能和可靠性优势。但它也存在一些局限性，尤其是当你更习惯于 SQL 等关系型数据库系统时。例如，Spark 无法确切知道某一行数据具体存在于哪台服务器上，因此无法直接选择特定行进行更新或删除。如果你已经习惯了这种数据库思维，就必须调整概念模型，将数据集视为一个整体，并基于列而非行来处理数据。

  • DataFrame：是由命名列组成的行集合
    • 结构上类似于 SQL 数据库，但属于非关系型
    • 不可变：DataFrame 在创建后无法被更改，但可以被转换为新的 DataFrame（结果是产生两个 DataFrame：原始 DataFrame 和转换后的 DataFrame）。数据集可以被覆盖，但 Foundry 会记录版本历史，方便你随时探索并回退到历史构建版本。
    • 惰性求值：一系列转换任务会作为一个单一（组合）操作进行求值，并在触发构建时执行。
  • 弹性分布式数据集：（RDD）是 DataFrame 的底层数据结构。通过将 DataFrame 划分为多个不相交的子集，转换操作可以在计算机集群（网络）中的多台计算机（节点）上并行求值。这些过程均在后台自动完成，但在编写 PySpark 代码时牢记这一点非常重要。
  • 共享变量：默认情况下，为了提高效率，Spark 会将转换任务中使用的每个变量单独管理的副本发送到每台并行计算机（节点）。如果必须在任务间共享变量，Spark 支持两种类型的共享变量：
    1. 广播变量：将值缓存（保存）在内存（RAM）中，并广播给集群中的所有计算机（节点）
    2. 累加器：是可以进行累加或聚合的变量，包括但不限于计数器和求和。此概念与 GroupedData 相关，常用于统计计算。
  • 为什么使用 DataFrame？：Spark DataFrame 经过专门设计与优化，用于处理大规模（PB 级以上）的结构化数据集合。
  • 为什么要编写 PySpark 代码？：PySpark 让你能够在 Code Repositories 和 Code Workbook 中定制数据集的转换方式，比单独使用 Contour 或 Blacksmith 实现更复杂、更灵活的操作。
  • PySpark 不适用于什么场景？：PySpark 专为转换数据集而设计，不适合用于访问单个数据值。你可以用它计算总和与平均值，但不应也不能直接引用底层数据。

SQL 查询会生成“视图”（虚拟表结果集），与之不同的是，使用 PySpark 处理数据集会生成全新的数据集。这不仅能让你基于派生数据集构建新数据集，组织内的其他成员也能复用这些中间数据集来完成各自的数据处理任务。Palantir Foundry 作为一个数据操作系统，会通过父子（或源-结果）有向树关系自动关联数据集。这种机制让任何人都能轻松追踪 Spark 转换的血缘关系。换言之，你可以探索数据集的依赖构建过程，以及这些数据集的来源。你还能发现组织内其他成员是如何使用某个数据集的，从而借鉴示例或有效减少重复工作。

理解 PySpark 代码

基础入门代码

在 Code Workbook 中，你的函数可能类似于以下形式：



已复制！
1def new_frame(old_frame):
2  df = old_frame
3  # df = transformations on df
4  return df
  • old_frame：引用存储在 Foundry 中的代表数据集的 DataFrame。old_frame 是不可变的，意味着它不能在这个 new_frame 函数内被修改。从某种意义上说，转换的每一个中间步骤都会生成一个新的、不可变的 DataFrame，你可以选择继续对其进行转换或原样返回。虽然实际情况并非完全如此，但作为一种认知模型，它有助于你更好地组织代码。
  • new_frame：你可以在此函数内定义一系列要应用于 old_frame 的转换操作。你的 return 语句应返回一个 DataFrame（在此示例中我们称之为 df）。在底层机制中，你应用于该 DataFrame 的所有转换操作都会被合并与优化，然后才会应用于输入数据集。一旦你使用自己的代码触发构建，结果将保存为 Foundry 中的一个新的数据集文件，构建完成后即可进行探索。

DataFrame 中的数据无法直接引用，因为它既不是 Array 也不是 Dictionary。实际上，由于底层一直在进行分区和混洗操作，在任何给定时刻都无法确定任何数据的具体位置。除非你要对数据集进行过滤或聚合，否则你编写的代码应该对数据集的具体内容保持相对透明。排序操作通常开销大且速度慢，因此经验法则是：假设每一行都是随机排序的，将你的工具手段限制在列操作、过滤、聚合以及发挥你自己的创造性解决问题的能力上。

警告

清楚掌握传入列的模式非常重要，因为 PySpark 不是类型安全的，它会尝试评估所有转换操作，并在运行期间任何操作失败时中断。

不要对字符串或日期执行数学函数，不要对数字执行字符串操作，也不要对整数执行日期操作，因为冲突类型的行为难以预测。

在操作之前，务必将值转换为正确的类型。

命名列

DataFrame 的每一列都有名称（且可重命名）。列名是唯一的且区分大小写。对于 Foundry 数据集，请遵循以下准则：

  • 始终使用小写字母或数字。
  • 使用 _（下划线）而不是空格来分隔单词（因为不允许使用空格）。
  • 按照约定，避免使用 camelCasedColumnNames（驼峰式命名）。
  • 切勿使用特殊字符，例如 ( 、 ) 或 &。
  • 聚合函数有时会自动为列名添加特殊字符。在转换中返回 DataFrame 之前，你需要提供别名或重命名该列。

链式转换

当你接手现有代码时，会发现对于如何命名引用 DataFrame 的变量并没有硬性规定。在本速查表中，DataFrame 将引用为 df，但在其他示例中可能是 raw、out、input、table、something_specific。只要能完成任务，怎么命名都可以。

你还会注意到这种模式：

已复制！
1df = df.select("firstName", "age")
2df = df.withColumn("age", df.age.cast("integer"))
3df = df.filter(df.age > 21)
4df = df.withColumnRenamed("firstName", "first_name")
5return df

或者（同样的逻辑，写法不同）：



已复制！
1return df.select("firstName", "age") \
2       .withColumn("age", df.age.cast("integer")) \
3       .filter(df.age > 21) \
4       .withColumnRenamed("firstName", "first_name")

如果你不熟悉代码：等号左侧的 df 用于存储对右侧 df 应用转换后的结果，然后再执行下一行代码。在这个例子中，我们将结果存入了同名的变量中，本质上是在每一步之后覆盖了 df 原有的内容。你也可以使用不同的名称来保存 DataFrame 转换的结果，但在大多数情况下，直接覆盖变量名并继续操作是没问题的。在每个转换函数的最后，我们必须返回新的 dataframe，既可以作为变量返回（如第一个示例所示），也可以作为最后一步转换的结果返回（如第二个示例所示）。

这两个示例实现的功能是一样的：

  1. 仅从 df 中选取我们希望在转换后数据集中包含的 2 列
  2. 将 age 列进行类型转换，确保它是整数而不是字符串。
  3. 过滤数据集的行，仅包含 age > 21 的记录
  4. 将列 firstName 重命名为 first_name

最终生成的数据集将只包含 first_name 和 age 这两列，且年龄在 21 岁及以下的人员已被排除在外。这就是 df 在最后所包含的内容，你可以将其 return 或对其应用更多的转换操作。我们将在接下来的章节中更详细地探讨这些转换操作。

在 Code Repositories 中，你必须在 .py 文档的顶部声明以下导入语句，才能使用大部分功能：
