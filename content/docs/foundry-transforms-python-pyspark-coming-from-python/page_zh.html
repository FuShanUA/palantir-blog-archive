从 Python 转型而来

如果你具备 Python 经验，可能已经习惯了以面向过程或命令式的方式处理数据：给出将数据从一种状态转换为另一种状态所需的精确步骤。SQL 则截然不同，它属于声明式语言，只需描述期望的结果，软件会负责生成该结果。PySpark 是一个旨在通过 Python 方便地构建复杂 SQL 查询的库：它试图以 Python 的过程式语法来提供对 SQL 概念的访问。这结合了 Python 的灵活性、SQL 的便捷性以及 Spark 的并行处理能力。

转变你的概念模型，从数据集整体的视角进行思考，并基于列而非行来处理数据，这会大有裨益。我们不再直接使用变量、列表、字典、循环等操作数据，而是围绕 DataFrame 展开。这意味着，我们不再使用 Python 原生类型和运算符，而是使用 Spark 内置的运算符，以分布式方式对 DataFrame 进行大规模处理。

示例

假设你在 Python 中有一个数字列表，想给每个数字加 5。

有趣的是，DataFrame 对象实际上并未将数据保存在内存中：它只是对 Spark 中数据的引用。DataFrame 采用了惰性求值机制。当我们要求 Spark 真正对 DataFrame 执行操作时（例如将其写入 Foundry），它会遍历我们创建的所有中间 DataFrame，生成优化后的查询计划，并在 Spark 集群上执行。这使得 Foundry 能够突破单台服务器或笔记本电脑内存容量的限制，实现弹性扩展。
