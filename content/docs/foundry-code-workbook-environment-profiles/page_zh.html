Code Workbook 配置文件

Code Workbook 配置文件是一组预定义的 Conda 包和 Spark 设置，可作为特定用例或用户群体的默认环境。针对给定的 Code Workbook 配置文件，你还可以配置预热模块，以减少用户等待时间。

例如，数据科学团队可能希望配置一个包含 tensorflow 和 keras 等 Conda 包的数据科学配置文件，并分配更多的驱动内存。

在 Code Workbook 中，用户可以在“环境配置”对话框中从一系列 Code Workbook 配置文件里进行选择。随后，用户可以进一步自定义 Conda 包。用户无法针对给定的 Code Workbook 配置文件自定义 Spark 设置。

针对给定的配置文件，其权限边界为项目。对配置文件拥有查看权限的用户可将该配置文件导入到项目中。配置文件导入项目后，项目中任何使用 Code Workbook 的用户均可使用该配置文件。

Artifacts 配置文件

在控制面板中新建的 Code Workbook 配置文件使用 Artifacts。基于 Artifacts 的配置文件允许使用由 Artifacts 安全生产的库，包括未发布到 shared 频道、在 Foundry 中编写的 Python 库。请参阅 控制面板文档，以详细了解如何创建 Artifacts 配置文件。

使用 Artifacts 配置文件

在控制面板中创建 Artifacts 配置文件后，用户即可在 Code Workbook 中使用该配置文件。Artifacts 配置文件包含所请求的包列表以及提供这些包的后端存储库列表。要使用该配置文件，必须将配置文件中的所有后端存储库作为项目导入项添加到工作簿所在的项目中。如果你没有权限导入配置文件中的所有后端存储库，你将无法使用该配置文件来设置环境。

如果配置文件的某个后端存储库在后续不再导入到项目中，你将无法获取环境，直至将其重新添加为导入项；系统会在界面中提示你执行此操作。

自定义 Artifacts 配置文件

在 Code Workbook 中使用自定义的 Artifacts 配置文件时，所使用的后端存储库列表属于该 Code Workbook，而非配置文件。新建的 Code Workbook 初始化时不包含任何后端存储库，随着用户在工作簿中使用自定义的 Artifacts 环境，后端存储库列表会自动填充。请注意，这意味着所有使用自定义 Artifacts 环境的用户（跨越工作簿中的所有分支）都在添加并使用同一个后端存储库列表。

在 Code Workbook 中使用自定义的 Artifacts 环境时，工作簿的所有后端存储库必须导入到项目中。如果工作簿的某个后端存储库在后续不再导入到项目中，你将无法获取环境，直至将其重新添加为导入项；系统会在界面中提示你执行此操作。

未来计划

未来，所有现有的配置文件和环境都将迁移至 Artifacts。这意味着无论基础配置文件为何，用户都能够使用由 Artifacts 安全生产的库。
