概念：用户自定义函数

用户自定义函数允许你在 PySpark 中使用任意自定义的 Python 代码。例如，你可以使用 UDF 从数据集每一行的复杂文本格式中解析信息。

声明之后，UDF 的工作方式与 PySpark 的内置函数（如 concat、date_diff、trim 等）类似。

动机

与直觉相反，在正常情况下，数据实际上并不会被引入到你的 Python 代码中。当你使用 PySpark 操作 DataFrame 时，你实质上是在描述 Spark 集群为了生成最终的 DataFrame 而应采取的分布式并行步骤。这使得 Spark 和 Foundry 能够实现几乎无限的扩展，但也引入了 UDF 这一少量设置工作，以便将代码注入集群并在实际数据上运行。PySpark 会将你的 UDF 代码发送到运行查询的每一台服务器上。

考虑不使用 UDF

相比于 Spark 优化后的内置功能，Python 带来的额外开销使得 UDF 的运行速度相对较慢。建议考虑使用 PySpark 的内置函数来表达你的逻辑。

1"Weather report: rain 55-62"

假设我们要从以下天气格式中获取最低温度，在此例中为 55。我们可以编写如下普通的 Python 函数：

1def extract_low_temperature(weather_report):
2	return int(weather_report.split(' ')[-1].split('-')[0])

我们将围绕函数 extract_low_temperature 创建一个 UDF，以将其集成到我们的 PySpark 查询中。创建 UDF 需要提供我们的函数及其在 PySpark 类型系统中的预期返回类型。



Copied!
1# Import the necessary type
2from pyspark.sql.types import IntegerType
3
4# Wrap our function as a UDF
5low_temp_udf = F.udf(extract_low_temperature, IntegerType())

现在，可以将该 UDF 用于 DataFrame，将整个列作为参数传入。

1df = df.withColumn('low', low_temp_udf(F.col('weather_report')))
