容器化转换

前提条件

以下文档假设您已具备容器化基础设施及容器镜像等概念的基础知识。如果您对这些主题不熟悉，建议您先查阅《Docker 概览文档 ↗》。

Foundry 通过两种方式与推送到平台中的容器进行交互：

  1. 使用转换 sidecar 装饰器。
  2. 构建并使用容器支持模型资产。

在这两种用例中，第一步都是在遵循下列镜像要求的同时，将镜像推送到 Foundry 内部托管的 Docker 注册表中。

镜像要求

  1. 镜像具有数字形式的 userID。
  • Dockerfile 中定义的 userID 必须是数字且不能为 '0'。Foundry 不允许以 root 用户身份在容器内运行命令；在某些系统中 '0' 会被解释为零，而非数字 ID 则可能被设定为以 root 身份运行。
  • 对于自带容器转换，userID 必须明确为 '5001'。
  1. 镜像基于 linux/amd64 平台构建。
  • Foundry 仅支持执行基于该平台构建的容器。Docker 的默认平台是 linux，因此在执行Docker 构建命令 ↗时，需添加 --platform linux/amd64 以明确指定平台。
  1. 推送镜像时需带有 digest 或除 latest 之外的任何标签。
  • 执行的 Docker 推送命令应指定镜像的 digest，或使用非 latest 的标签。Foundry 不会执行标记为 latest 的镜像，因为没有任何机制能确保特定镜像始终是最新版本。
  1. 最大镜像层大小需小于约 10 GB。
  • 强烈建议每个镜像层大小保持在 10 GB 以下。如果您的使用场景需要更大的镜像层，请联系您的 Palantir 代表。
  1. 任何暴露的端口均须介于 1024 和 65535 之间。
  • 0 至 1023 端口属于公认端口，因此保留给 root 使用。Foundry 不允许以 root 用户身份在容器内运行命令；因此，在 Foundry 中启动镜像时，指定在该范围内的任何端口均无法使用。
  1. [可选] 镜像已启用遥测。
  • 若要从您的容器中启用遥测日志记录：
    • 镜像必须在 /bin/sh 路径下包含 shell 可执行文件。
    • 镜像必须支持 set 和 tee shell 命令。
