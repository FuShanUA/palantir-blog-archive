我们的影响力

HHS Protect

Foundry 作为 Palantir 平台的一种配置，在 COVID-19 疫情期间协助 HHS、CDC、FEMA、DOD、NIH 和 FDA 等机构进行跨部门的大流行管理协调。

我们的能力

Foundry 旨在解决碎片化格局带来的一些关键挑战。

  • 数据集成与协调
  • 数据共享
  • 数据访问
  • 数据质量
  • 数据驱动工作流
  • 互操作性
  • 数据隐私与安全

数据集成与协调

灵活地收集、集成和协调数据

Foundry 可集成并协调复杂的多模态数据，使其随时可用于分析。其安全的开放架构、API 和动态本体支持：

  • 将数据安全地集成、转换和协调到一个开放环境中
  • 与任何系统（包括 EHR/EMR）的双向连接
  • 实现以前无法并排使用的数据（包括 HL7、FHIR、CDISC 数据）的互操作性
  • 实现数据模型（FHIR 和 OMOP）与术语标准（LOINC、SNOMED、MedDRA）的互操作性

数据共享

安全地共享数据

借助开放的 API 和安全可扩展的环境，Foundry 能够促进跨系统和跨机构的大规模安全数据共享，从而实现：

  • 多个合作伙伴之间的无缝协作与安全数据共享
  • 在数据访问控制范围内，具备不同技能水平的用户之间开展协作

数据访问

动态管理数据访问

Foundry 的功能使数据主人——从机构到其合作伙伴——能够根据标准、政策和法规集中管理数据。

  • Foundry 的细粒度访问控制、安全标记和多层级组织结构在平台内全面生效，确保用户仅拥有其被授权的数据访问级别。
  • Foundry 的控制机制十分灵活——允许根据需要快速重新配置权限、角色等设置。

数据质量

管理数据质量

借助 Foundry，用户可以验证数据和分析结果是否准确、高质量且最新。Foundry 通过以下方式实现对复杂数据的有效管理：

  • 数据协调
  • 自动数据健康检查
  • 数据质量与隐私验证
  • 用于识别和解决质量问题的工具
  • 资源版本控制
  • 数据溯源追踪
  • 对代码和数据进行分支管理，以保护主环境

这些功能有助于生成模型就绪数据，允许建模用户检查数据、了解其有效性并识别偏差。

数据驱动工作流

促进研究与运营工作流

Foundry 提供的工具能够契合用户的实际使用习惯。其应用近实时地从数据环境中提取数据并回写，以支持连通的工作流和知识管理。Foundry 提供：

  • 用于队列构建和分析的点击式工具
  • 支持开放语言的基于代码的工具
  • 一个模型管理系统，用于测试、部署和管理模型——所有这些都可以直接基于实时的生产数据完成

互操作性

防止供应商锁定

Foundry 具备开放性和互操作性，可防止供应商锁定。

  • 某美国主要政府机构通过测试在第三方平台提取和复制 Foundry 数据、元数据和逻辑的能力，验证了 Foundry 的开放性。
  • 该机构认定 Foundry 具备开放性和互操作性，其能力确立了互操作性的标准，该机构所有其他 IT 投资项目均以此标准进行评估。

数据隐私与安全

保护数据隐私与安全

隐私和安全是我们工作的核心价值，直接内置于我们的软件之中。

  • Foundry 是一款获得 FedRAMP Moderate 授权的现代 SaaS 平台，可满足 SOC 2 Type 2、FISMA High、HIPAA 及其他标准。
  • 我们提供安全处理 PII、PHI 和其他敏感数据的功能，包括用于准备公开访问数据集的数据保护工具。
  • Foundry 中的所有数据始终由数据主人控制和所有，绝不归 Palantir Technologies 所有。
