我们的能力

Foundry 旨在解决由碎片化环境引发的一些关键挑战。

数据集成与协调

灵活地收集、集成和协调数据

Foundry 集成并协调复杂的多模态数据，使其随时可用于分析。其安全的开放架构、API 以及动态本体支持以下功能：

  • 将数据安全地集成、转换和协调到一个开放环境中
  • 与任何系统（包括 EHR/EMR）进行双向连接
  • 实现先前无法并行使用的数据（包括 HL7、FHIR、CDISC 数据）的互操作性
  • 实现数据模型（FHIR 和 OMOP）以及术语（LOINC、SNOMED、MedDRA）的互操作性

数据共享

安全地共享数据

借助开放的 API 和安全、可扩展的环境，Foundry 能够促进跨系统和跨机构的大规模安全数据共享，从而实现：

  • 多个合作伙伴之间的无缝协作和安全数据共享
  • 在数据访问控制范围内，具备不同技能组合的用户之间开展协作

数据访问

动态统筹数据访问

Foundry 的功能使数据主人——从机构到其合作伙伴——能够根据标准、政策和法规集中治理数据。

  • Foundry 的细粒度访问控制、安全标记和多层级组织结构在整个平台中层层传递，以确保用户仅拥有其被授权的数据访问级别。
  • Foundry 的控制机制十分灵活——允许根据需要快速重新配置权限、角色等。

数据质量

管理数据质量

借助 Foundry，用户可以验证数据和分析是否准确、高质量且保持最新。Foundry 通过以下方式使管理数据复杂性成为可能：

  • 数据协调
  • 自动化数据健康检查
  • 数据质量和隐私校验
  • 用于识别和解决质量问题的工具
  • 资源版本控制
  • 数据溯源跟踪
  • 对代码和数据进行分支管理，以保护主环境

这些功能促进了可用于建模的数据——允许建模用户检查数据、了解其有效性并识别偏见。

数据驱动的工作流

促进研究和运营工作流

Foundry 提供了能够适应用户当下工作方式的工具。其应用程序近乎实时地从数据环境中提取数据并回写，以支持连通的工作流和知识管理。Foundry 提供：

  • 用于队列构建和分析的点击式工具
  • 支持开放语言的基于代码的工具
  • 一个模型管理系统，用于测试、部署和管理模型——所有这些都直接建立在实时生产数据之上

互操作性

防止供应商锁定

Foundry 是开放且可互操作的，能够防止供应商锁定。

  • 美国一家主要政府机构验证了 Foundry 的开放性，测试了在第三方平台中提取和复制 Foundry 数据、元数据和逻辑的能力。
  • 该机构认为 Foundry 具有开放性和互操作性，其能力树立了互操作性的标准，该机构所有其他的 IT 投资项目均以此标准进行评估。

数据隐私与安全

保护数据隐私和安全

隐私和安全是我们工作的核心价值观，直接内置于我们的软件中。

  • Foundry 是一种获得 FedRAMP Moderate 授权的现代 SaaS，可满足 SOC 2 Type 2、FISMA High、HIPAA 等标准。
  • 我们提供安全处理 PII、PHI 和其他敏感数据的功能，包括用于准备公共访问数据集的数据保护工具。
  • Foundry 中的所有数据始终由数据主人控制和拥有——绝不由 Palantir Technologies 拥有。
