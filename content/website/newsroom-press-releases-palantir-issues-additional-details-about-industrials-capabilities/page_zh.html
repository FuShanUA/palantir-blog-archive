Palantir 公布“双击”工业版演示中待展示功能的更多细节

丹佛--(BUSINESS WIRE)--Palantir Technologies (NYSE:PLTR) 今日公布了其将在 2021 年 4 月 14 日星期三美国东部时间上午 11:00 举行的“双击”工业版演示中重点展示的 Palantir Foundry 功能的更多细节。

Palantir 在工业领域的客户涵盖汽车、分销、重型制造、化工和公用事业等行业。这些客户依赖 Foundry 优化运营与维护、预测网络健康状况，并应对突发事件。

在 Palantir “双击”系列活动的首期，工程师将演示实现上述工作的最新功能：

  • 链式模型仿真。 Foundry 协调跨所有组织领域的数据模型，从而在物联网时间序列数据与支撑性元数据之间建立语义关联。随后，该本体可作为镜像组织实际运营的模型输入源，其输出结果则回传至本体中。为理解决策的全面影响，可将这些模型串联在一个数字孪生中，用于实时适应事件或对未来事件进行严密的仿真。通过将模型构建与操作统一于单一平台，任何洞见和优化都会渗透至组织的日常决策中。

  • 无与伦比的分析计算能力。 针对本体与时间序列数据，往往需要在“复杂但缓慢”与“简单但快速”的查询间进行取舍。为克服这一固有矛盾，Foundry 的查询优化器会重写分析查询，将算子下推至“热”存储和“冷”存储的数据层。这意味着查询执行性能更佳，资源占用更小。此外，无论查询是通过 API 提交、来自模型，还是经由 Foundry 的图形分析应用发起，优化器均对其适用。这使客户能够实时查询多达数万亿条传感器读数。

  • 毫无取舍的动态编排。 Foundry 提供所有已接入平台的物联网时间序列数据。通过终端用户 API 和各类工具，数据唾手可得，且无需将所有内容存入高性能存储而产生额外成本。传统时间序列系统迫使客户在两者间取舍：要么将数据保存在优化但昂贵的“热”存储中，要么将归档数据卸载至较廉价的“冷”存储，但后者将无法再进行查询。Foundry 的“热/温”存储系统消除了这一矛盾，可按需将归档数据自动提取至热存储中。借助 Foundry，客户在机器生成数据上实现了高达 50 倍的压缩比（具体取决于初始格式）。这使得资本资产配备数千个传感器的工业客户（如飞机、油井和制造工厂）能够每秒流式传输并接入数百万个新数据点。

除上述功能外，Palantir 还将重点介绍基于 Foundry 构建的两款全新原型方案，让工业客户只需点击几下即可应用上述能力，包括：

  • Foundry 质量原型方案 使客户能够快速识别新出现的问题，加速根因分析，管理预测性维护，降低保修成本并提高召回准确率，以及在实施高昂的生产或供应链变更之前，对更换原材料或提高产量等改动进行数字仿真。

  • Foundry 供应链原型方案 使客户能够预测网络健康状况，针对潜在问题生成预警提醒，通过算法比较并优化供应链场景，应对持续的冲击，并监控成本、准时率和二氧化碳排放等关键绩效指标。
